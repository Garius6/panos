# AGENTS.md

## О проекте

Panos — экспериментальный интерпретатор самописного языка программирования,
реализованный на Odin. Язык русскоязычный по ключевым словам и синтаксически
ориентирован на смесь Go и Rust.

Текущий pipeline:

1. `lexer.odin` — токенизация исходного текста.
2. `parser.odin` — построение AST.
3. `resolver.odin` — связывание имен с символами.
4. `type_cheker.odin` — проверка и вывод типов.
5. `compiler.odin` — компиляция AST в байткод.
6. `vm.odin` — выполнение байткода.
7. `main.odin` — запуск файла и диагностический вывод всех стадий.

`test.ps` — основной демонстрационный файл языка. `e2e_test.odin` содержит
сквозные тесты pipeline через `run_code`.

## Команды

Запуск тестового файла:

```sh
odin run . -debug -vet -strict-style -vet-tabs -warnings-as-errors -- test.ps
```

То же через Justfile:

```sh
just debug-file test.ps
```

Автотесты:

```sh
odin test . -debug -vet -strict-style -vet-tabs -warnings-as-errors
```

Тесты могут печатать предупреждения memory tracker о текущих утечках
аллокатора. Если команда завершилась с кодом `0`, тест считается успешным.

## Язык Panos

Ключевые слова и базовый синтаксис:

- `функ имя(арг: Тип) -> Тип ... конец` — функция.
- `функ(арг: Тип) -> Тип ... конец` — лямбда.
- `пер имя = значение` — локальная переменная.
- `возврат значение` — возврат.
- `если условие тогда ... иначе ... конец` — if-выражение.
- `пока условие цикл ... конец` — цикл.
- `тип Имя = структура ... конец` — структура.
- `тип Имя = интерфейс ... конец` — интерфейс.
- `реализация Тип ... конец` — методы структуры.
- `реализация Интерфейс для Тип ... конец` — реализация интерфейса.

Типы:

- `Число`, `Булево`, `Строка`, `Пусто`
- туплы: `(Число, Строка)`
- функции: `функ(Число, Строка) -> Булево`
- массивы: `Массив(Число)`
- соответствия/map: `Соответствие(Строка, Число)`

Функции, методы интерфейсов, методы структур и лямбды требуют явных типов
параметров и возвращаемого значения. Универсального типа-заглушки в языке нет.
Пользовательские структуры и интерфейсы типизируются номинально: структуры
совместимы с интерфейсами только через явную `реализация Интерфейс для Тип`.

Коллекции:

```panos
пер числа: Массив(Число) = массив()
числа.добавить(1)
числа[0] = 10
пер x = числа.получить(0, 0)

пер цены = соответствие(
    "яблоко" = 10,
    "груша" = 20,
)
цены["банан"] = 5
пер есть_банан = цены.есть("банан")
```

Поддержанные методы массивов:

- `.длина()`
- `.добавить(значение)`
- `.получить(индекс, иначе)`
- `.есть(индекс)`
- `.содержит(значение)`

Поддержанные методы соответствий:

- `.длина()`
- `.есть(ключ)`
- `.получить(ключ, иначе)`
- `.удалить(ключ)`

Индексирование `[]` строгое: отсутствующий ключ map или индекс массива вне
границ должен приводить к runtime error. Метод `.получить(..., иначе)` нужен
для безопасного чтения.

## Рабочие соглашения

- Сохранять русскоязычные сообщения ошибок и диагностический вывод.
- Предпочитать минимальные вертикальные срезы: lexer → parser → resolver →
  type checker → compiler → VM → e2e-тест.
- При добавлении синтаксиса сначала покрывать его сквозным тестом в
  `e2e_test.odin`, если это практически возможно.
- Не коммитить `panos.dSYM/` и другие локальные артефакты сборки.
- Не менять unrelated untracked файлы без явного запроса.

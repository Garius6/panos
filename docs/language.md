# Язык Panos

Panos — экспериментальный русскоязычный язык с синтаксисом, близким к Go и
Rust. Реализация сейчас оптимизирована под небольшие вертикальные срезы:
лексер, парсер, resolver, type checker, compiler и VM развиваются вместе.

Документ описывает текущее поведение языка, а не желаемый финальный дизайн.

## Программа

Программа состоит из объявлений верхнего уровня. Точка входа — функция
`старт`.

```panos
функ старт() -> Число
    42
конец
```

Функция возвращает значение последнего выражения тела. Можно использовать
явный `возврат`.

```panos
функ удвоить(x: Число) -> Число
    возврат x * 2
конец
```

## Комментарии и кодировка

Исходники пишутся в UTF-8. Ключевые слова, стандартные типы и большая часть
диагностик русскоязычные. Документированные примеры используют русский стиль
именования, но идентификаторы не обязаны быть русскими.

## Базовый синтаксис

### Переменные

```panos
пер x = 10
пер имя: Строка = "Panos"
```

Аннотация типа у локальной переменной необязательна, если type checker может
вывести тип из выражения. Переменная не может иметь тип `Пусто`.

### Арифметика и сравнение

```panos
пер a = 10 + 5 * 2
пер b = a > 12
пер c = "pan" + "os"
```

`+` поддерживает числа и строки. Арифметические операторы работают с числами.
Сравнения возвращают `Булево`.

### Строки

Строковые литералы записываются в двойных кавычках. Поддержаны escape-
последовательности:

- `\n` — перевод строки.
- `\t` — табуляция.
- `\r` — возврат каретки.
- `\"` — двойная кавычка внутри строки.
- `\\` — обратный слэш.

```panos
пер новая_строка = "\n"
пер таб = "\t"
пер пример = "ключ = \"значение\"\n"
```

### If-выражение

```panos
если x > 0 тогда
    "плюс"
иначе
    "ноль или минус"
конец
```

`если` является выражением. Если есть `иначе`, ветки должны иметь совместимые
типы. Ветка с `паника(...)` имеет тип `Никогда` и совместима с обычной веткой.

### Цикл

```panos
пер i = 0
пока i < 10 цикл
    i = i + 1
конец
```

Цикл используется для побочных эффектов и возвращает `Пусто`. Внутри цикла
можно использовать `продолжить`, чтобы перейти к следующей проверке условия, и
`прервать`, чтобы выйти из цикла.

```panos
пер сумма = 0
пер i = 0

пока i < 10 цикл
    i = i + 1

    если i == 3 тогда
        продолжить
    конец

    если i > 5 тогда
        прервать
    конец

    сумма = сумма + i
конец
```

`для` — сахар, разворачивается в `пока` на этапе разбора. Форма по
итерируемому значению (нужен `.длина()` + `[индекс]`, например `Массив`):

```panos
для элемент в массив цикл
    ио.печать(элемент)
конец

для (ключ, значение) в соответствие.записи() цикл
    ио.печать(ключ)
конец
```

Числовой диапазон, границы включительны с обеих сторон:

```panos
для сч = 0 по 4 цикл
    ио.печать(сч)
конец
```

## Типы

Базовые типы:

- `Число`
- `Булево`
- `Строка`
- `Пусто`

Составные типы:

- тупл: `(Число, Строка)`
- функция: `функ(Число, Строка) -> Булево`
- массив: `Массив(Число)`
- соответствие: `Соответствие(Строка, Число)`
- опция: `Опция(Число)`
- результат: `Результат(Число, Ошибка)`

Внутренний тип `Никогда` используется для выражений, которые не возвращают
обычное значение, например `паника(...)`. Пользовательский тип-заглушка для
явных аннотаций не поддержан.

## Функции и лямбды

Обычные функции требуют явных типов параметров и возвращаемого значения.

```panos
функ сложить(a: Число, b: Число) -> Число
    a + b
конец
```

Лямбда записывается через `функ(...) ... конец`. Типы параметров и результата
могут выводиться из ожидаемого типа.

```panos
функ применить(ф: функ(Число) -> Число, значение: Число) -> Число
    ф(значение)
конец

функ старт() -> Число
    применить(функ(x)
        x + 1
    конец, 41)
конец
```

Если ожидаемый тип функции неизвестен, лямбде может потребоваться больше
контекста, иначе type checker остановится с ошибкой вывода.

## Структуры

Структура объявляется через `тип ... = структура`.

```panos
тип Точка = структура
    x: Число
    y: Число
конец

функ старт() -> Число
    пер p = Точка(10, 20)
    p.x + p.y
конец
```

Конструктор структуры вызывается по имени типа и принимает значения всех полей
в порядке объявления.

## Методы структур

Методы объявляются в блоке `реализация Тип`.

```panos
тип Счетчик = структура
    значение: Число
конец

реализация Счетчик
    функ увеличить(self: Счетчик, шаг: Число) -> Число
        self.значение + шаг
    конец
конец
```

Первый аргумент метода должен иметь тип структуры-получателя. Вызов метода
записывается через точку.

## Интерфейсы

Интерфейс объявляет набор методов.

```panos
тип Печатаемый = интерфейс
    функ текст(self: Печатаемый) -> Строка
конец
```

Структура совместима с интерфейсом только через явную реализацию:

```panos
реализация Печатаемый для Документ
    функ текст(self: Документ) -> Строка
        self.имя
    конец
конец
```

Типизация структур и интерфейсов номинальная, не структурная.

## Перечисления (ADT)

Пользовательский алгебраический тип объявляется через `тип X = перечисление ...
конец`. Каждый вариант имеет имя и опциональный список типов полей:

```panos
тип Фигура = перечисление
    Точка
    Круг(Число)
    Прямоугольник(Число, Число)
конец
```

Правила:

- Хотя бы один вариант обязателен.
- Имена вариантов уникальны в пределах типа.
- Вариант без полей записывается без скобок (`Точка`); вариант с полями — со
  скобками (`Круг(Число)`). Пустые `()` запрещены.
- `экспорт тип X = перечисление ... конец` экспортирует и тип, и все его
  варианты (правило «все или ничего»).
- Имя варианта не должно совпадать с другим символом модуля (структурой,
  интерфейсом, функцией).
- Тип рекурсивен: вариант может ссылаться на объявляемый тип
  (`тип Дерево = перечисление Лист; Узел(Дерево, Дерево) конец`).

### Конструкторы

Значение создаётся вызовом конструктора (или использованием имени без скобок
для варианта без полей). Тип аргументов должен точно совпадать с объявленным:

```panos
пер точка = Точка
пер круг = Круг(3)
пер прямо = Прямоугольник(4, 5)
```

Когда короткое имя варианта неоднозначно (несколько ADT в области видимости с
одноимённым вариантом), используется квалификация `Тип.Вариант`:

```panos
пер к = Фигура.Круг(3)
```

Для импортированного модуля работает `модуль.Тип.Вариант(...)`. Форма
`модуль.Вариант(...)` допустима, только если короткое имя однозначно среди
экспортов модуля.

## Выражение `выбор`

`выбор` разбирает значение суммы (пользовательский ADT, `Опция(T)` или
`Результат(T, E)`) по вариантам. Значение выражения — общий супертип тел
веток (аналогично `если`).

```panos
функ площадь(ф: Фигура) -> Число
    возврат выбор ф
        Точка -> 0
        Круг(р) -> р * р * 314 / 100
        Прямоугольник(ш, в) -> ш * в
    конец
конец
```

Поддерживаемые шаблоны:

- `_` — заглушка, совпадает с любым значением. Может быть только последней
  веткой.
- Идентификатор-биндер — привязывает значение subject целиком к имени; тоже
  должен быть последней веткой.
- Конструктор `Имя(поле1, поле2, ...)` — совпадает с этим вариантом. Каждый
  аргумент шаблона — `_` или биндер поля с соответствующим типом.
- Квалифицированный конструктор: `Тип.Вариант(...)` или `модуль.Тип.Вариант(...)`.

Type checker гарантирует исчерпывающий разбор: если ни `_` ни биндер-ветка не
покрывают все варианты, программа отклоняется со списком непокрытых имён.
Дублирующиеся ветки для одного варианта и ветки после `_` — тоже ошибка
компиляции.

Для встроенных `Опция(T)` и `Результат(T, E)` разбор работает единообразно:

```panos
пер о: Опция(Число) = Опция.Есть(41)
пер удвоенное = выбор о
    Есть(х) -> х * 2
    Нет -> 0
конец
```

В шаблонах `выбор` конструктор варианта можно писать и без квалификации
(`Есть(х)`), и с ней (`Опция.Есть(х)`) — ограничение «только через
`Тип.Вариант`» (Стадия 18) касается конструктора-ВЫРАЖЕНИЯ, а не шаблона:
шаблон резолвит имя варианта по ожидаемому типу subject'а, а не через
scope.

## Массивы

```panos
пер числа: Массив(Число) = массив()
числа.добавить(10)
числа[0] = 20
```

Поддержанные методы:

- `.длина()` — количество элементов.
- `.добавить(значение)` — добавить элемент.
- `.получить(индекс, иначе)` — безопасное чтение.
- `.есть(индекс)` — проверка индекса.
- `.содержит(значение)` — поиск значения.

Индексирование `[]` строгое: выход за границы — runtime error.

## Соответствия

```panos
пер цены = соответствие(
    "яблоко" = 10,
    "груша" = 20,
)

цены["банан"] = 5
```

Поддержанные методы:

- `.длина()` — количество пар.
- `.есть(ключ)` — проверка ключа.
- `.получить(ключ, иначе)` — безопасное чтение.
- `.удалить(ключ)` — удалить ключ.

Ключи соответствия должны иметь допустимый тип ключа. Строгое индексирование
отсутствующего ключа приводит к runtime error.

## Дженерики

Функции, структуры, перечисления и их методы могут быть параметризованы
типом через `[T]`/`[A, B]` после имени.

Лямбды получают полиморфизм ранга 1 неявно, без `[T]` — параметры и
результат выводятся из ожидаемого типа контекста:

```panos
функ применить(ф: функ(Число) -> Число, значение: Число) -> Число
    ф(значение)
конец
```

Обычные функции параметризуются явно:

```panos
функ первый[T](а: T, б: T) -> T
    а
конец
```

Структуры:

```panos
тип Пара[A, B] = структура
    первый: A
    второй: B
конец

пер p: Пара(Число, Строка) = Пара(1, "один")
```

Перечисления (в том числе рекурсивные):

```panos
тип Дерево[T] = перечисление
    Лист
    Узел(T, Дерево(T), Дерево(T))
конец
```

Методы объявляются в `реализация Тип` БЕЗ `[T]` в заголовке блока — целевой
тип уже generic по своей декларации:

```panos
реализация Пара
    функ поменять(это: Пара) -> Пара
        Пара(это.второй, это.первый)
    конец
конец
```

Метод может добавить СВОЙ, независимый от владельца, type-параметр:

```panos
реализация Пара
    функ с_первым[C](это: Пара, новый: C) -> Пара(C, B)
        Пара(новый, это.второй)
    конец
конец
```

Ограничения:

- `реализация Интерфейс для Generic-структура` не поддержана — generic-типы
  не могут реализовывать интерфейсы.
- Bounded generics (`T: Сравнимый` и т.п.) не поддержаны — параметр типа
  ничем не ограничен, кроме структурной применимости внутри тела.
- Explicit instantiation (`первый[Число](1, 2)`) не поддержан — типы всегда
  выводятся из аргументов или ожидаемого типа.

`Опция(T)`/`Результат(T, E)` (ниже) — не встроенные примитивы, а обычные
generic-перечисления, объявленные тем же синтаксисом в стандартной
«прелюдии», неявно доступной каждому модулю без `импорт`.

## Опция

`Опция(T)` описывает значение, которое может отсутствовать.

```panos
пер a: Опция(Число) = Опция.Есть(10)
пер b: Опция(Число) = Опция.Нет()
```

Конструкторы:

- `Опция.Есть(значение)` — значение присутствует.
- `Опция.Нет()` — значения нет.

Методы:

- `.есть()` — `истина`, если значение присутствует.
- `.пусто()` — `истина`, если значения нет.
- `.значение()` — строгое извлечение; runtime error для `Опция.Нет()`.
- `.получить(иначе)` — значение или значение по умолчанию.
- `.ожидать("сообщение")` — значение или runtime panic.
- `.результат_или(ошибка)` — `Опция.Есть(T)` в `Результат.Успех(T)`, `Опция.Нет()` в
  `Результат.Неудача(ошибка)`.
- `.заменить_значение(значение)` — заменить значение только у `Есть`
  (`Нет` остаётся `Нет`). Собственный type-параметр метода — новое
  значение может быть другого типа, чем исходное (`Опция(Число)` →
  `Опция(Строка)`).
- `.запас(другая_опция)` — вернуть текущую `Есть`, иначе запасную `Опция`.

Пример:

```panos
функ старт() -> Число
    пер о: Опция(Число) = Опция.Нет()
    о.запас(Опция.Есть(42)).получить(0)
конец
```

## Результат

`Результат(T, E)` описывает обычную ошибку без аварийного завершения.

```panos
пер a: Результат(Число, Ошибка) = Результат.Успех(10)
пер b: Результат(Число, Ошибка) = Результат.Неудача(Ошибка("фс", "нет файла"))
```

Конструкторы:

- `Результат.Успех(значение)` — успешное значение.
- `Результат.Неудача(ошибка)` — обычная ошибка.

Стандартный тип ошибки создается так:

```panos
Ошибка("код", "сообщение")
```

Методы:

- `.успех()` — `истина` для `Успех`.
- `.ошибка()` — `истина` для `Неудача`.
- `.значение()` — строгое извлечение значения; runtime error для `Неудача`.
- `.причина()` — строгое извлечение ошибки; runtime error для `Успех`.
- `.получить(иначе)` — успешное значение или значение по умолчанию.
- `.получить_ошибку(иначе)` — ошибка или ошибка по умолчанию.
- `.ожидать("сообщение")` — успешное значение или runtime panic.
- `.ожидать_ошибку("сообщение")` — ошибка или runtime panic.
- `.опция()` — `Результат.Успех(T)` в `Опция.Есть(T)`, `Результат.Неудача(E)` в `Опция.Нет()`.
- `.ошибка_опция()` — `Результат.Неудача(E)` в `Опция.Есть(E)`, `Результат.Успех(T)` в `Опция.Нет()`.
- `.заменить_значение(значение)` — заменить значение только у `Успех`
  (`Неудача` остаётся `Неудача`); может сменить тип `T`.
- `.заменить_ошибку(ошибка)` — заменить ошибку только у `Неудача`
  (`Успех` остаётся `Успех`); может сменить тип `E`.
- `.запас(другой_результат)` — вернуть текущий `Успех`, иначе запасной
  `Результат`.

Пример:

```panos
функ прочитать() -> Результат(Строка, Ошибка)
    Результат.Неудача(Ошибка("фс", "нет файла"))
конец

функ старт() -> Строка
    прочитать().получить("пусто")
конец
```

## Оператор `?`

`?` разворачивает `Опция` или `Результат`.

Для `Опция(T)`:

- `Опция.Есть(value)?` дает `value`.
- `Опция.Нет()?` досрочно возвращает `Опция.Нет()` из текущей функции.

```panos
функ получить() -> Опция(Число)
    Опция.Нет()
конец

функ добавить_один() -> Опция(Число)
    Опция.Есть(получить()? + 1)
конец
```

Для `Результат(T, E)`:

- `Результат.Успех(value)?` дает `value`.
- `Результат.Неудача(error)?` досрочно возвращает `Результат.Неудача(error)` из текущей функции.

```panos
функ получить() -> Результат(Число, Ошибка)
    Результат.Неудача(Ошибка("данные", "нет значения"))
конец

функ добавить_один() -> Результат(Число, Ошибка)
    Результат.Успех(получить()? + 1)
конец
```

`?` можно использовать только внутри функции с совместимым возвращаемым типом.
Для `Результат` тип ошибки пробрасываемого значения должен совпадать с типом
ошибки функции.

## Panic

`паника("сообщение")` используется для критичных ошибок.

```panos
функ старт() -> Число
    паника("невозможно продолжить")
конец
```

Panic не является обычным `Результат`. Это аварийный путь выполнения. Type
checker моделирует `паника(...)` как `Никогда`, поэтому выражение можно
использовать там, где обычная ветка возвращает другой тип:

```panos
функ старт() -> Число
    если истина тогда 42 иначе паника("невозможно") конец
конец
```

## Модули

Модули подключаются через `импорт`.

```panos
импорт математика

функ старт() -> Число
    математика.квадрат(6)
конец
```

Экспортируемые объявления помечаются ключевым словом `экспорт`.

```panos
экспорт функ квадрат(x: Число) -> Число
    x * x
конец
```

Resolver и type checker проверяют наличие импортированного модуля и
экспортируемого символа.

## Индексирование и присваивание

Индексирование поддержано для массивов, соответствий и строк:

```panos
числа[0]
цены["яблоко"]
строка[0]
```

Присваивание поддержано для переменных, полей и индексируемых мест там, где VM
реализует соответствующую операцию.

```panos
x = 10
точка.x = 5
числа[0] = 99
```

## Ошибки type checker и runtime

Panos различает:

- `Type Error` — ошибка статической проверки.
- `Runtime Error` — ошибка выполнения обычной операции, например строгого
  индексирования вне границ.
- `Runtime Panic` — критичный сбой через `паника(...)` или `.ожидать(...)`.

Сообщения ошибок должны оставаться русскоязычными.

## Зарезервированные решения

- Не использовать `.или(...)` как имя метода. `или` зарезервировано под будущий
  логический оператор или ключевое слово языка.
- Для fallback-методов контейнеров используется имя `.запас(...)`.
- Полноценные transform-комбинаторы над `Опция` и `Результат` пока не
  документированы как поддержанные: для них нужно отдельно развивать вызов
  функций из VM-методов коллекций.

# Рецепты

Рецепты — пошаговые инструкции для типичных изменений интерпретатора. В
отличие от глав [Pipeline](../lexer.md)/[Рантайм](../memory-and-gc.md)/
[LSP](../lsp.md), рецепт **не объясняет архитектуру заново** — он
ссылается на уже написанные главы и компонует их в последовательность
действий. Если рецепт вводит новый архитектурный факт, которого нет в
соответствующей главе — это повод сначала дополнить главу, а не факт в
рецепте (см. известные ограничения ниже).

## Формат рецепта

Каждый рецепт следует одной структуре:

```markdown
# Рецепт: <название изменения>

## Файлы по порядку (pipeline order)

<Нумерованный список файлов в порядке прохождения pipeline — от лексера к
VM/LSP, не в произвольном порядке.>

## Шаги

<Для каждого файла — что именно добавить/изменить: МЕСТО и ФОРМА правки,
без готового кода-решения. Рецепт учит находить место, не копирует чужой
diff.>

## Проверка

<Конкретная команда(ы), подтверждающая успех.>
```

## Список рецептов

- [Новый бинарный оператор](./new-binary-operator.md) — затрагивает лексер,
  парсер, тайпчекер, компилятор и VM.
- [Новый LSP-метод](./new-lsp-method.md) — по образцу уже реализованных
  методов `lsp/lsp_server.odin`.
- [Новая stdlib-функция](./new-stdlib-function.md) — экспортируемая функция
  в существующем модуле `std/*.ps`.
- [Новый вид AST-узла](./new-ast-node.md) — новое выражение или statement.
- [Новая diagnostic-проверка](./new-diagnostic.md) — резолвер или тайпчекер.
- [Новый stdlib-модуль](./new-stdlib-module.md) — целиком новый файл
  `std/*.ps`.

Список открытый — новые рецепты добавляются по мере появления типичных
задач, без пересмотра структуры этого раздела.

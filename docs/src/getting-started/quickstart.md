# Быстрый старт

## Hello world локально

Точка входа любой программы — функция `старт`. Она возвращает значение
последнего выражения тела:

```panos
функ старт() -> Число
    10 + 20
конец
```

Сохраните это в `hello.ps` и запустите:

```sh
./panos hello.ps
```

```text
30
```

Явный `возврат` работает как обычно, если нужен ранний выход (этот и все
следующие примеры в документации можно редактировать и запускать прямо на
странице — кнопка «▶ Запустить»):

```panos
функ проверить(x: Число) -> Строка
    если x < 0 тогда
        возврат "отрицательное"
    конец
    "неотрицательное"
конец

функ старт() -> Строка
    проверить(-5)
конец
```

## Hello world в браузере

Ставить ничего не нужно — [веб-песочница](https://garius6.github.io/panos/playground/)
компилирует и исполняет panos прямо в браузере (тот же компилятор и VM,
собранные в WASM). Там же по умолчанию открыт `test.ps` — файл-пример,
который использует практически все конструкции языка сразу: структуры,
интерфейсы, дженерики, ADT и `выбор`, `Опция`/`Результат`, циклы, замыкания.
Хороший способ посмотреть, «как это всё выглядит вместе», не читая
документацию по главам.

Ограничения браузерной версии (в отличие от нативного CLI):

- Нет `импорт` — файловый импорт модулей недоступен в браузере (нет ФС).
  Значит недоступны и модули, которые сами требуют импорта (`ввод_вывод`,
  `фс`, `сеть`, `строки`, `математика` и т.п.) — только то, что встроено в
  язык напрямую (арифметика, `Опция`/`Результат`, методы массивов/карт).
- Нет ввода со stdin — `ввод_вывод.прочитать_строку` (и он всё равно был бы
  недоступен из-за пункта выше) в браузере работать не может в принципе:
  WASM-вызов синхронный, а у браузера нет блокирующего stdin.

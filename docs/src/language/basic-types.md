# Базовые типы и операторы

## Кодировка и идентификаторы

Исходники — UTF-8. Ключевые слова, стандартные типы и большая часть
диагностик компилятора русскоязычные. Идентификаторы (имена переменных,
функций, типов) не обязаны быть русскими — можно смешивать латиницу и
кириллицу как удобно, компилятору всё равно.

## Зарезервированные слова

Единая hard-reserved политика: ничего из зарезервированной лексики нельзя
использовать как имя переменной/функции/типа/параметра — ни глобально, ни
локально. Два тира отличаются только МЕХАНИЗМОМ проверки, не строгостью:

- **Жёсткие ключевые слова лексера** — отдельные токены, `Ident`-токен
  для них в принципе не порождается: `пер`, `конст`, `истина`, `ложь`,
  `функ`, `возврат`, `конец`, `пока`, `цикл`, `продолжить`, `прервать`,
  `если`, `тогда`, `иначе`, `не`, `и`, `или`, `тип`, `структура`,
  `реализация`, `интерфейс`, `для`, `в`, `импорт`, `экспорт`,
  `перечисление`, `выбор`, `как`, `запусти`. Попытка использовать любое
  из них как идентификатор — синтаксическая ошибка на этапе парсинга
  ("ожидается идентификатор").
- **Reserved-имена builtin-функций** — обычные `Ident`-токены на уровне
  лексера, но резолвер отклоняет ЛЮБУЮ попытку объявить символ с таким
  именем (top-level функцию/тип, локальную `пер`/`конст`, параметр
  функции/лямбды, деструктурированное/for-in имя, pattern-биндер в
  `выбор`) — `получить`, `отправить`, `себя`, `паника`, `наблюдать`,
  `получить_сигнал`, `длина`, `Ошибка`. `пер получить = 5` — Resolve
  Error "зарезервированное имя, нельзя использовать", так же как и
  top-level `функ получить() -> ... конец`.

Разный механизм — лексер против резолвера — потому что `в` не заводит
никакого символа даже при обычном использовании (чистая грамматическая
частица в `для x в коллекция`, синтаксическая позиция целиком статична),
а builtin-функции — настоящие вызываемые значения с реальным поведением
в рантайме. Наблюдаемый эффект для пишущего код один и тот же: слово из
списка выше нельзя переопределить нигде.

## Переменные

```panos
функ старт() -> Строка
    пер x = 10
    конст имя: Строка = "panos"
    имя
конец
```

`пер` объявляет обычную (переприсваиваемую) переменную, `конст` —
неизменяемый биндинг: `имя = "другое"` после `конст имя = ...` даёт
ошибку типизации ("попытка переприсвоить константу"). Оба — полноценные
объявления переменной, различие только в возможности переприсвоения.
`конст` — **binding-immutability**, не глубокая неизменяемость: если
`имя` ссылается на структуру, её поля (`имя.поле = ...`) всё равно можно
менять — у panos структуры reference semantics (см. [«Структуры и
методы»](structs-and-methods.md)), `конст` фиксирует только сам биндинг,
не то, на что он указывает.

Аннотация типа (`: Тип`) необязательна для обеих форм, если type checker
может вывести тип из выражения инициализации. Переменная **не может**
иметь тип `Пусто` — это ловится как ошибка на этапе типизации, не в
рантайме.

Shadowing (повторное объявление того же имени, `пер`/`конст` в любой
комбинации) запрещено в ТОМ ЖЕ scope ("Имя x уже объявлено"), но
разрешено во ВЛОЖЕННОМ scope — внутренний биндинг временно затеняет
внешний, снаружи блока видно исходное значение.

## Базовые типы

- `Число` — число с плавающей точкой (f64).
- `Целое` — целое число. На рантайме то же f64-представление, что и
  `Число` (нет отдельного целочисленного значения в VM) — различие
  целиком на уровне типов, проверяется компилятором.
- `Булево` — `истина` / `ложь`.
- `Строка`.
- `Пусто` — тип отсутствия значения (аналог `void`); не может быть типом
  переменной.
- `Никогда` — тип выражений, которые никогда не возвращают обычное значение
  (например, `паника(...)`); нет синтаксиса для явной аннотации этим типом,
  он появляется только выведенным.

## Число и Целое

Числовой литерал (`42`, `0`, `-1`) по умолчанию всегда `Число` — точка в
записи роли не играет (`42` и `42.0` типизируются одинаково). Чтобы
получить `Целое`, нужен явный Целое-контекст — аннотация переменной,
тип поля структуры, тип параметра функции и т.п.:

```panos
функ старт() -> Целое
    пер a: Целое = 5      // литерал в Целое-контексте — сам становится Целое
    пер b: Целое = a + 3  // 3 рядом с Целое-операндом — тоже Целое
    b
конец
```

Между уже типизированными значениями `Число` и `Целое` **нет неявной
коэрсии** — `Число`-переменную нельзя использовать там, где ожидается
`Целое`, и наоборот, без явного преобразования. Явное преобразование —
`Целое(x)`/`Число(x)` (конструктор-синтаксис, как у структур/enum'ов, а
не отдельный оператор `as`): `Число->Целое` усекает дробную часть к нулю
(та же семантика, что у `Целое/Целое`-деления), `Целое->Число` точное,
оба направления — no-op при вызове на уже своём типе.

```panos
пер n: Число = 3.7
пер i = Целое(n)     // 3 (усечение к нулю, не округление)
пер обратно = Число(i)
```

`/` над двумя `Целое` — целочисленное деление с усечением к нулю (как в
C/Rust/Go, не флор-деление Python): `-7 / 2` даёт `-3`, не `-4`. `%` —
остаток от такого деления, определён **только** для `Целое` (у `Число`
остатка нет — для float-модуло есть `математика` в стандартной
библиотеке, если понадобится).

`Целое` обязателен везде, где число используется как индекс или размер:
индекс массива/строки (`arr[i]`), `длина(...)`/`.длина()`, числовой
диапазон в `для i = A по B цикл` (счётчик `i` — `Целое`). Это ловит
классический баг "случайно проиндексировал float-ом" на этапе
компиляции, а не в рантайме.

## Арифметика и сравнение

```panos
функ старт() -> Строка
    пер a = 10 + 5 * 2      // 20
    пер b = a > 12          // истина
    пер c = "pan" + "os"    // "panos"
    c
конец
```

`+ - * /` — только числа, кроме `+`, который дополнительно работает как
конкатенация строк. Возведение в степень как оператор не поддержано —
для степени есть `математика.степень` в стандартной библиотеке. Остаток
(`%`) поддержан для `Целое` (см. выше).

Сравнения: `< > <= >= == <>` (обратите внимание — «не равно» это `<>`, а не
привычный `!=`). Первые четыре числовые: `<`/`>`/`<=`/`>=` не определены для
строк, структур, массивов и т.п. (кроме типов, реализующих `Сравниваемое`
— см. [«Интерфейсы прелюдии»](./prelude-interfaces.md)).

## Битовые операторы

`& | ^ ~ << >>` — как в C/Rust/Go, только для `Целое` (та же политика,
что у `%` выше — нет смысла в поразрядных операциях над `Число`):

```panos
функ старт() -> Целое
    пер флаги: Целое = 0
    пер бит: Целое = 3
    флаги = флаги | (1 << бит)  // выставить бит 3
    флаги & ~(1 << бит)         // сбросить бит 3
конец
```

Приоритет (от слабого к сильному): `|` < `^` < `&` < `<<`/`>>` — то же
относительное расположение, что в C (сдвиг между аддитивными и
реляционными, `&`/`^`/`|` между равенством и логическими и/или), но без
самой путаницы C (там `&` связывает слабее `==`, частый источник ошибок
без скобок в реальном C-коде) — здесь порядок такой же по замыслу, но
раз таблица приоритетов переразмечена с нуля под этот язык, смешивать
битовые операторы с сравнениями/логикой в одном выражении без скобок
всё равно не рекомендуется.

## Логические операторы

```panos
функ старт() -> Булево
    (истина и ложь) или (не ложь)
конец
```

`и`/`или` — инфиксные ключевые слова (не `&&`/`||`), `не` — префиксный
унарный оператор. `и` связывает крепче `или` и слабее сравнений: `a == b и
c == d` читается как `(a == b) и (c == d)`, без скобок вокруг сравнений.

> Odin-рантайм печатает булевы значения как `true`/`false` (например, в
> выводе примеров на этой странице) — это формат вывода нативной сборки,
> не часть синтаксиса языка. Внутри самого panos-кода булевы литералы
> всегда `истина`/`ложь`.

## Строки

Двойные кавычки, стандартный набор escape-последовательностей:

- `\n` — перевод строки
- `\t` — табуляция
- `\r` — возврат каретки
- `\"` — кавычка внутри строки
- `\\` — обратный слэш

```panos
функ старт() -> Строка
    "ключ = \"значение\"\n"
конец
```

Индексирование строки по числу возвращает односимвольную строку:

```panos
функ старт() -> Строка
    "panos"[0]
конец
```

### Байты

`длина(...)`/`.длина()`/`строки.срез` работают по РУНАМ (символам), не
байтам — корректно для кириллицы и любого текста, но непригодно для
бинарных данных (двоичное содержимое файла — не обязано быть валидным
UTF-8, случайные байты декодировались бы как руны неправильно). Для
такого — отдельные байтовые примитивы в `строки`:

```panos
импорт строки

функ старт() -> Целое
    пер s = "abc"
    строки.байт(s, 0)          // 97 ('a') — Целое 0..255, не рун
конец
```

`строки.байт(s, i) -> Целое`, `строки.длина_байт(s) -> Целое`,
`строки.срез_байт(s, начало, конец) -> Строка` (границы — байтовые
смещения, не рунные), `строки.из_байтов(Массив(Целое)) -> Строка`
(каждый элемент — 0..255). Хранилище `Строка` и так byte-transparent
(это обычный байтовый буфер под капотом — `фс.прочитать` не проверяет
UTF-8), только рунные операции (`длина`/`срез`/индексация) на бинарных
данных дают неверный результат. Байтовые примитивы — то, на чём построен
`std/архив.ps` (tar; см. [«Стандартная библиотека»](./standard-library.md#архив-tar)).

### Интерполяция

`\(выражение)` внутри строкового литерала (Swift-style) — вставляет
результат выражения прямо по месту:

```panos
функ старт() -> Строка
    пер имя = "Гайдар"
    пер возраст = 30
    "Привет, \(имя)! Тебе \(возраст) лет."
конец
```

Выражение внутри `\( )` может быть любым (в т.ч. с вложенными скобками,
вызовами, другой строкой со своей интерполяцией) и не обязано быть
`Строка` заранее — оно неявно проходит через тот же builtin `встроку(x)`,
который использует `ввод_вывод.печать`: числа/булевы/строки печатаются
как есть, структуры/перечисления с `реализация Печатаемое` — через
`.вСтроку()`, иначе — позиционный structural dump (см.
[«Интерфейсы прелюдии»](./prelude-interfaces.md#печатаемое)). Экранировать
саму интерполяцию (получить буквальные символы `\(`) — двойной слэш:
`"\\(не_интерполяция)"`.

`встроку(x)` вызываем и напрямую, отдельно от интерполяции — тот же
конвертер значения в `Строка`, что и внутри `\( )`.

## Время

```panos
импорт время

функ старт() -> Число
    пер начало = время.монотонно_мс()
    // ... какая-то работа ...
    время.монотонно_мс() - начало
конец
```

`время.монотонно_мс() -> Число` — миллисекунды с момента старта VM,
монотонные тики (не подвержены переводу системных часов) — для
измерения ИНТЕРВАЛОВ (например, sliding-window лимита рестартов в
[`std/супервизор.ps`](./processes.md#супервизия)). `время.сейчас_мс() ->
Число` — unix-время в миллисекундах, wall-clock — для отображения/
логов, НЕ для измерения интервалов (перевод часов ломает монотонность,
используйте `монотонно_мс` для этого). Оба builtin-модуля (как `строки`/
`фс`/`ос`), не `std/*.ps`-библиотека. Часов, календаря, таймзон,
форматирования дат в panos нет — только миллисекунды с двух эпох.

## Туплы

```panos
функ старт() -> Число
    пер позиция = (10, 25)
    позиция.0 + позиция.1
конец
```

Тип — `(Число, Число)`. Доступ к полям по позиции через `.0`, `.1` и т.д.,
без именования полей. Тупл можно и разобрать целиком на отдельные
биндинги — `пер (a, b) = позиция`, см. [«Структуры и методы» →
деструктуризация](./structs-and-methods.md#деструктуризация).

- массив: `Массив(Число)` — глава [«Массивы и соответствия»](./collections.md)
- соответствие (карта): `Соответствие(Строка, Число)` — там же
- опция: `Опция(Число)` — глава [«Опция и Результат»](./option-and-result.md)

# Базовые типы и операторы

## Кодировка и идентификаторы

Исходники — UTF-8. Ключевые слова, стандартные типы и большая часть
диагностик компилятора русскоязычные. Идентификаторы (имена переменных,
функций, типов) не обязаны быть русскими — можно смешивать латиницу и
кириллицу как удобно, компилятору всё равно.

## Переменные

```panos
пер x = 10
пер имя: Строка = "panos"
```

`пер` — единственный способ объявить переменную. Аннотация типа (`: Тип`)
необязательна, если type checker может вывести тип из выражения
инициализации. Переменная **не может** иметь тип `Пусто` — это ловится
как ошибка на этапе типизации, не в рантайме.

## Базовые типы

- `Число` — единственный числовой тип (нет отдельных int/float).
- `Булево` — `истина` / `ложь`.
- `Строка`.
- `Пусто` — тип отсутствия значения (аналог `void`); не может быть типом
  переменной.
- `Никогда` — тип выражений, которые никогда не возвращают обычное значение
  (например, `паника(...)`); нет синтаксиса для явной аннотации этим типом,
  он появляется только выведенным.

## Арифметика и сравнение

```panos
пер a = 10 + 5 * 2      // 20
пер b = a > 12          // истина
пер c = "pan" + "os"    // "panos"
```

`+ - * /` — только числа, кроме `+`, который дополнительно работает как
конкатенация строк. Остаток (`%`) и возведение в степень (`**`/`^`) как
операторы не поддержаны — для степени есть `математика.степень` в
стандартной библиотеке.

Сравнения: `< > <= >= == <>` (обратите внимание — «не равно» это `<>`, а не
привычный `!=`). Все шесть числовые: `<`/`>`/`<=`/`>=` не определены для
строк, структур, массивов и т.п.

## Логические операторы

```panos
пер a = истина и ложь        // ложь
пер b = ложь или истина      // истина
пер c = не ложь              // истина
```

`и`/`или` — инфиксные ключевые слова (не `&&`/`||`), `не` — префиксный
унарный оператор. `и` связывает крепче `или` и слабее сравнений: `a == b и
c == d` читается как `(a == b) и (c == d)`, без скобок вокруг сравнений.

> Odin-рантайм печатает булевы значения как `true`/`false` (например, через
> `fmt.println` в CLI) — это формат вывода нативной сборки, не часть
> синтаксиса языка. Внутри самого panos-кода булевы литералы всегда
> `истина`/`ложь`.

## Строки

Двойные кавычки, стандартный набор escape-последовательностей:

- `\n` — перевод строки
- `\t` — табуляция
- `\r` — возврат каретки
- `\"` — кавычка внутри строки
- `\\` — обратный слэш

```panos
пер пример = "ключ = \"значение\"\n"
пер первая_буква = "panos"[0]  // "p"
```

Индексирование строки по числу возвращает односимвольную строку.

## Тупы

```panos
пер позиция = (10, 25)
пер x = позиция.0
пер y = позиция.1
```

Тип — `(Число, Число)`. Доступ к полям по позиции через `.0`, `.1` и т.д.,
без именования полей.

## Остальные составные типы

Встречаются по мере знакомства с языком — здесь только имена для справки:

- функция: `функ(Число, Строка) -> Булево` — глава [«Функции»](./functions.md)
- массив: `Массив(Число)`
- соответствие (карта): `Соответствие(Строка, Число)`
- опция: `Опция(Число)`
- результат: `Результат(Число, Ошибка)`

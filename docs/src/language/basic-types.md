# Базовые типы и операторы

## Кодировка и идентификаторы

Исходники — UTF-8. Ключевые слова, стандартные типы и большая часть
диагностик компилятора русскоязычные. Идентификаторы (имена переменных,
функций, типов) не обязаны быть русскими — можно смешивать латиницу и
кириллицу как удобно, компилятору всё равно.

## Переменные

```panos
функ старт() -> Строка
    пер x = 10
    конст имя: Строка = "panos"
    имя
конец
```

`пер` объявляет обычную (переприсваиваемую) переменную, `конст` —
неизменяемый биндинг: `имя = "другое"` после `конст имя = ...` даёт
ошибку типизации ("попытка переприсвоить константу"). Оба — полноценные
объявления переменной, различие только в возможности переприсвоения.
`конст` — **binding-immutability**, не глубокая неизменяемость: если
`имя` ссылается на структуру, её поля (`имя.поле = ...`) всё равно можно
менять — у panos структуры reference semantics (см. [«Структуры и
методы»](structs-and-methods.md)), `конст` фиксирует только сам биндинг,
не то, на что он указывает.

Аннотация типа (`: Тип`) необязательна для обеих форм, если type checker
может вывести тип из выражения инициализации. Переменная **не может**
иметь тип `Пусто` — это ловится как ошибка на этапе типизации, не в
рантайме.

Shadowing (повторное объявление того же имени, `пер`/`конст` в любой
комбинации) запрещено в ТОМ ЖЕ scope ("Имя x уже объявлено"), но
разрешено во ВЛОЖЕННОМ scope — внутренний биндинг временно затеняет
внешний, снаружи блока видно исходное значение.

## Базовые типы

- `Число` — число с плавающей точкой (f64).
- `Целое` — целое число. На рантайме то же f64-представление, что и
  `Число` (нет отдельного целочисленного значения в VM) — различие
  целиком на уровне типов, проверяется компилятором.
- `Булево` — `истина` / `ложь`.
- `Строка`.
- `Пусто` — тип отсутствия значения (аналог `void`); не может быть типом
  переменной.
- `Никогда` — тип выражений, которые никогда не возвращают обычное значение
  (например, `паника(...)`); нет синтаксиса для явной аннотации этим типом,
  он появляется только выведенным.

## Число и Целое

Числовой литерал (`42`, `0`, `-1`) по умолчанию всегда `Число` — точка в
записи роли не играет (`42` и `42.0` типизируются одинаково). Чтобы
получить `Целое`, нужен явный Целое-контекст — аннотация переменной,
тип поля структуры, тип параметра функции и т.п.:

```panos
функ старт() -> Целое
    пер a: Целое = 5      // литерал в Целое-контексте — сам становится Целое
    пер b: Целое = a + 3  // 3 рядом с Целое-операндом — тоже Целое
    b
конец
```

Между уже типизированными значениями `Число` и `Целое` **нет неявной
коэрсии** — `Число`-переменную нельзя использовать там, где ожидается
`Целое`, и наоборот, без явного преобразования. Явный cast между ними
пока не реализован.

`/` над двумя `Целое` — целочисленное деление с усечением к нулю (как в
C/Rust/Go, не флор-деление Python): `-7 / 2` даёт `-3`, не `-4`. `%` —
остаток от такого деления, определён **только** для `Целое` (у `Число`
остатка нет — для float-модуло есть `математика` в стандартной
библиотеке, если понадобится).

`Целое` обязателен везде, где число используется как индекс или размер:
индекс массива/строки (`arr[i]`), `длина(...)`/`.длина()`, числовой
диапазон в `для i = A по B цикл` (счётчик `i` — `Целое`). Это ловит
классический баг "случайно проиндексировал float-ом" на этапе
компиляции, а не в рантайме.

## Арифметика и сравнение

```panos
функ старт() -> Строка
    пер a = 10 + 5 * 2      // 20
    пер b = a > 12          // истина
    пер c = "pan" + "os"    // "panos"
    c
конец
```

`+ - * /` — только числа, кроме `+`, который дополнительно работает как
конкатенация строк. Возведение в степень (`**`/`^`) как оператор не
поддержано — для степени есть `математика.степень` в стандартной
библиотеке. Остаток (`%`) поддержан для `Целое` (см. выше).

Сравнения: `< > <= >= == <>` (обратите внимание — «не равно» это `<>`, а не
привычный `!=`). Первые четыре числовые: `<`/`>`/`<=`/`>=` не определены для
строк, структур, массивов и т.п. (кроме типов, реализующих `Сравниваемое`
— см. [«Интерфейсы прелюдии»](./prelude-interfaces.md)).

## Логические операторы

```panos
функ старт() -> Булево
    (истина и ложь) или (не ложь)
конец
```

`и`/`или` — инфиксные ключевые слова (не `&&`/`||`), `не` — префиксный
унарный оператор. `и` связывает крепче `или` и слабее сравнений: `a == b и
c == d` читается как `(a == b) и (c == d)`, без скобок вокруг сравнений.

> Odin-рантайм печатает булевы значения как `true`/`false` (например, в
> выводе примеров на этой странице) — это формат вывода нативной сборки,
> не часть синтаксиса языка. Внутри самого panos-кода булевы литералы
> всегда `истина`/`ложь`.

## Строки

Двойные кавычки, стандартный набор escape-последовательностей:

- `\n` — перевод строки
- `\t` — табуляция
- `\r` — возврат каретки
- `\"` — кавычка внутри строки
- `\\` — обратный слэш

```panos
функ старт() -> Строка
    "ключ = \"значение\"\n"
конец
```

Индексирование строки по числу возвращает односимвольную строку:

```panos
функ старт() -> Строка
    "panos"[0]
конец
```

## Тупы

```panos
функ старт() -> Число
    пер позиция = (10, 25)
    позиция.0 + позиция.1
конец
```

Тип — `(Число, Число)`. Доступ к полям по позиции через `.0`, `.1` и т.д.,
без именования полей. Тупл можно и разобрать целиком на отдельные
биндинги — `пер (a, b) = позиция`, см. [«Структуры и методы» →
деструктуризация](./structs-and-methods.md#деструктуризация).

# Массивы и соответствия

## Массивы

```panos
функ старт() -> Целое
    пер числа: Массив(Целое) = массив(1, 2, 3)
    числа.добавить(4)
    числа[0] = числа[0] + 9
    числа[0] + числа.получить(2, 0) + числа.длина()
конец
```

Тип — `Массив(T)` для любого элементного типа `T`. Методы: `.длина()`
(возвращает `Целое`), `.добавить(значение)`, `.получить(индекс,
запасное)` (безопасный доступ, не паникует на выходе за границы),
`.есть(значение)`, `.содержит(значение)`. Индексирование `[]` — и для
чтения, и для записи, паникует при выходе за границы (в отличие от
`.получить`). Индекс — всегда `Целое` (см. [«Базовые типы» → Число и
Целое](./basic-types.md#число-и-целое)).

## Соответствия (карты)

```panos
функ старт() -> Целое
    пер цены: Соответствие(Строка, Целое) = соответствие(
        "яблоко" = 10,
        "груша" = 20,
    )
    цены["банан"] = 5
    цены.удалить("груша")
    цены.получить("банан", 0) + цены.длина()
конец
```

Тип — `Соответствие(K, V)`. Методы: `.длина()` (возвращает `Целое`),
`.есть(ключ)`, `.получить(ключ, запасное)`, `.удалить(ключ)`. `[]` —
чтение/запись по ключу, паникует, если ключа нет (используйте
`.получить` для безопасного чтения). Карта не индексируется позиционно —
перебор только через `.записи()` (см. [«Управляющие
конструкции»](./control-flow.md#для)), которая даёт тупы `(ключ,
значение)`.

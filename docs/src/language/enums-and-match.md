# Перечисления (ADT) и `выбор`

## Варианты

Вариант без полей, с одним полем, с несколькими полями — можно смешивать
в одном перечислении:

```panos
тип Фигура = перечисление
    Точка
    Круг(Число)
    Прямоугольник(Число, Число)
конец

функ старт() -> Число
    пер ф: Фигура = Фигура.Прямоугольник(3, 4)
    выбор ф
        Точка -> 0
        Круг(р) -> р * р * 314 / 100
        Прямоугольник(ш, в) -> ш * в
    конец
конец
```

`выбор` обязан покрыть все варианты (или иметь `_` — wildcard-ветку).
Непокрытый вариант — ошибка компиляции, не рантайма.

```panos
тип Фигура = перечисление
    Точка
    Круг(Число)
    Прямоугольник(Число, Число)
конец

функ старт() -> Строка
    пер ф: Фигура = Фигура.Круг(10)
    выбор ф
        Точка -> "точка"
        Круг(_) -> "круг"
        _ -> "другое"
    конец
конец
```

Конструктор варианта без полей вызывается по имени, без скобок (`Точка`,
не `Точка()`).

## `Опция` и `Результат` — тоже перечисления

`Опция(T)`/`Результат(T, E)` из прелюдии — обычные generic-перечисления,
разбираются `выбор` так же, как пользовательский ADT:

```panos
функ разобрать(о: Опция(Число)) -> Число
    выбор о
        Опция.Есть(х) -> х
        Нет -> -1
    конец
конец

функ старт() -> Число
    разобрать(Опция.Есть(8))
конец
```

Подробнее про `Опция`/`Результат`, их методы и оператор `?` — в
[отдельной главе](./option-and-result.md).

## Литеральные шаблоны

`выбор` работает не только на перечислениях — на `Число`/`Строка`/
`Булево` тоже, с ветками-литералами:

```panos
функ описать(x: Число) -> Строка
    выбор x
        1 -> "один"
        2 -> "два"
        _ -> "другое"
    конец
конец

функ старт() -> Строка
    описать(1) + " " + описать(99)
конец
```

Для `Число`/`Строка` домен значений неперечислим — компилятор не может
доказать, что вы покрыли всё, поэтому финальная ветка `_` (или биндер,
как в примере ниже) **обязательна**. `Булево` — исключение: у него ровно
два значения, и `истина`/`ложь` вместе дают настоящую исчерпывающую
проверку без `_`:

```panos
функ старт() -> Строка
    выбор истина
        истина -> "да"
        ложь -> "нет"
    конец
конец
```

Ветка-биндер (без `_`, просто имя) ловит "всё остальное" и связывает в
него подошедшее значение — как и биндер на перечислении:

```panos
функ разобрать(s: Строка) -> Число
    выбор s
        "да" -> 1
        "нет" -> 0
        остальное -> -1
    конец
конец
```

Литеральные шаблоны работают и внутри конструктора варианта, ограничивая
конкретное поле:

```panos
тип Событие = перечисление
    Клик(Число, Число)
конец

функ старт() -> Строка
    пер e = Событие.Клик(1, 2)
    выбор e
        Событие.Клик(1, y) -> "клик по x=1"
        Событие.Клик(x, y) -> "обычный клик"
    конец
конец
```

Отрицательные числовые литералы (`-1 -> ...`) как шаблон не
поддерживаются — грамматика неоднозначна на границе строк с бинарным
вычитанием.

## Конструктор-шаблоны на структурах

`Тип(...)` работает не только на вариантах перечисления — на структурах
тоже, с теми же под-шаблонами (литерал, `_`, биндер, вложенный
конструктор), разобранными по полям в порядке их объявления:

```panos
тип Точка = структура
    x: Число
    y: Число
конец

функ классифицировать(п: Точка) -> Строка
    выбор п
        Точка(0, 0) -> "начало координат"
        Точка(0, y) -> "на оси Y"
        Точка(x, 0) -> "на оси X"
        Точка(_, _) -> "где-то ещё"
    конец
конец

функ старт() -> Строка
    классифицировать(Точка(0, 5))
конец
```

У структуры, в отличие от перечисления, нет вариантов — ветка `Тип(...)`,
где ВСЕ под-шаблоны `_`/биндер (как `Точка(_, _)` выше), покрывает
структуру целиком и действует как catch-all — обязана быть последней,
как и голый `_`. Ветка с частично сужающими под-шаблонами (`Точка(0,
0)`) сама по себе ничего не покрывает — без финальной catch-all ветки
компилятор требует её явно, той же логикой, что и для `Число`/`Строка`
выше.

Проверка исчерпываемости рекурсивна: вложенный конструктор-под-шаблон
засчитывается как покрывающий, если сам исчерпывающий для своего типа —
не только `_`/биндер на верхнем уровне:

```panos
тип Точка = структура
    x: Число
    y: Число
конец

тип Событие = перечисление
    Клик(Точка)
    Скролл(Число)
конец

функ старт() -> Строка
    пер e: Событие = Событие.Клик(Точка(1, 2))
    выбор e
        Событие.Клик(Точка(_, _)) -> "клик"
        Событие.Скролл(_) -> "скролл"
    конец
конец
```

Здесь `Точка(_, _)` покрывает вообще все точки, поэтому ветка `Событие.
Клик(...)` целиком закрывает вариант `Клик` — финальная catch-all
ветка компилятору не нужна, несмотря на два варианта у `Событие`.
Если бы под-шаблон был частичным (`Точка(1, _)`) — потребовалась бы
явная catch-all-ветка, как обычно.

## Именованные поля в структурных шаблонах

У структур (не у enum-вариантов — у тех нет имён полей, только
позиционные типы) можно указывать поля по имени вместо позиции —
`Тип(имя: под-шаблон, ...)`:

```panos
тип Точка = структура
    x: Число
    y: Число
конец

функ классифицировать(п: Точка) -> Строка
    выбор п
        Точка(x: 0, y: 0) -> "начало координат"
        Точка(y: 0) -> "на оси X"
        Точка(x: 0) -> "на оси Y"
        Точка(_, _) -> "где-то ещё"
    конец
конец
```

Именованная форма может быть ЧАСТИЧНОЙ — неупомянутые поля трактуются
как неявный `_` и не сужают покрытие этой ветки (в примере выше
`Точка(y: 0)` эквивалентно `Точка(x, 0)` из предыдущего раздела).
Позиционную и именованную форму нельзя смешивать в одном шаблоне —
либо все поля по позиции, либо все указанные явно по имени.

# Ввод-вывод и файловая система

> Всё на этой странице требует `импорт ввод_вывод`/`импорт фс` и реального
> локального окружения — **недоступно** в браузерной песочнице (подробнее
> в [«Модули» → ограничение в браузере](./modules.md#ограничение-в-браузере)).
> Примеры не запускаются на странице, запускайте локально: `./panos файл.ps`.

## `ввод_вывод`: печать и стандартный ввод

```panos-cli
импорт ввод_вывод

функ старт() -> Пусто
    ввод_вывод.печать("без перевода строки")
    ввод_вывод.строка("с переводом строки")
конец
```

`.печать(значение)`/`.строка(значение)` принимают значение **любого**
типа: если он реализует [`Печатаемое`](./prelude-interfaces.md#печатаемое),
используется `.вСтроку()`, иначе — structural-дамп полей.

```panos-cli
импорт ввод_вывод

функ старт() -> Строка
    ввод_вывод.прочитать_строку()
конец
```

`ввод_вывод.прочитать_строку() -> Результат(Строка, Ошибка)` читает одну
строку из stdin. Для нескольких строк подряд эффективнее
`ввод_вывод.поток()` — он даёт объект с тем же интерфейсом, что и файл
(см. ниже), так что `.прочитать_строку()` на нём можно вызывать в цикле,
не открывая stdin заново на каждый вызов.

## `фс`: файлы целиком

```panos-cli
импорт фс

функ старт() -> Строка
    фс.записать("заметки.txt", "первая строка\n")
    пер результат = фс.прочитать("заметки.txt")
    если результат.успех() тогда
        результат.значение()
    иначе
        "не удалось прочитать: " + результат.причина().сообщение
    конец
конец
```

- `фс.есть(путь) -> Булево`
- `фс.прочитать(путь) -> Результат(Строка, Ошибка)` — весь файл целиком
- `фс.записать(путь, содержимое) -> Результат(Число, Ошибка)` —
  перезаписывает файл целиком (создаёт, если его не было), число —
  количество записанных байт

## Файловые дескрипторы: построчная работа

Когда файла целиком мало — например, нужно писать по частям или читать
построчно — `фс.открыть(путь)` даёт дескриптор (создаёт файл, если его не
было; открывает на чтение и запись):

```panos-cli
импорт фс

функ старт() -> Строка
    пер файл = фс.открыть("лог.txt").ожидать("не удалось открыть лог")
    файл.записать("событие 1\n")
    файл.записать("событие 2\n")
    файл.закрыть()

    пер снова = фс.открыть("лог.txt").ожидать("не удалось открыть лог")
    пер первая_строка = снова.прочитать_строку().получить("(пусто)")
    снова.закрыть()
    первая_строка
конец
```

Методы дескриптора (тот же набор что у `ввод_вывод.поток()`):

- `.прочитать() -> Результат(Строка, Ошибка)` — всё оставшееся содержимое
- `.прочитать_строку() -> Результат(Строка, Ошибка)` — одна строка
- `.записать(текст) -> Результат(Число, Ошибка)`
- `.закрыть() -> Пусто`

Вызов метода на уже закрытом дескрипторе не паникует — возвращает
`Результат.Неудача(Ошибка(...))`, как любая другая I/O-ошибка.

## `ос`: переменные окружения и аргументы

```panos-cli
импорт ос

функ старт() -> Строка
    ос.окружение("HOME").получить("(не задано)")
конец
```

- `ос.окружение(имя) -> Опция(Строка)`
- `ос.установить_окружение(имя, значение) -> Результат(Число, Ошибка)`
- `ос.удалить_окружение(имя) -> Булево`
- `ос.аргументы() -> Массив(Строка)` — аргументы командной строки,
  переданные скрипту (не считая имени файла и флагов самого `panos`), см.
  [«Использование CLI»](../getting-started/cli.md).

# Функции

## Докстринги (`///`)

Обычный `// комментарий` лексер отбрасывает целиком — его текст никак не
доступен ни компилятору, ни инструментам. `///` (три слэша) — особый
случай: если такой комментарий стоит **вплотную** (без пустой строки)
над `функ`/`тип`, его текст прикрепляется к декларации и появляется в
hover-подсказке LSP при наведении на имя функции/типа:

```panos
/// Складывает два числа и возвращает сумму.
экспорт функ сложить(a: Число, b: Число) -> Число
    a + b
конец
```

Несколько подряд идущих строк `///` объединяются в один многострочный
докстринг. Пустая строка между докстрингом и декларацией (или между
двумя блоками `///`) рвёт привязку — такой комментарий остаётся обычным,
никуда не прикреплённым текстом:

```panos
/// Этот текст НЕ станет докстрингом — дальше пустая строка.

функ f() -> Целое
    1
конец
```

Работает для функций, структур, интерфейсов, перечислений и методов
внутри `реализация ... конец`. На компиляцию/типизацию докстринг не
влияет — это чисто информационная side-таблица (см. `Token.doc` в
`core/lexer.odin`), читаемая только LSP.

## Объявление

```panos
функ сложить(a: Число, b: Число) -> Число
    a + b
конец

функ старт() -> Число
    сложить(3, 4)
конец
```

Типы параметров и возвращаемого значения у именованной функции **всегда
обязательны** — `-> Тип` нельзя опустить, даже если функция ничего
осмысленного не возвращает (тогда пишется `-> Пусто`). Тело — это
последовательность выражений/операторов; значение последнего выражения и
есть результат функции.

```panos-cli
импорт ввод_вывод

функ записать_лог(сообщение: Строка) -> Пусто
    ввод_вывод.печать(сообщение)
конец

функ старт() -> Пусто
    записать_лог("привет")
конец
```

> Этот пример использует `импорт` — стандартная библиотека недоступна в
> браузере (подробнее в [разделе про модули](./modules.md)), поэтому блок
> не запускается прямо на странице. Запустите локально: `./panos файл.ps`.

Значение с типом, отличным от `Пусто`, в конце `Пусто`-функции просто
отбрасывается — не обязательно явно приводить последнее выражение к
`Пусто`.

## Именованные аргументы

Вызов может передавать аргументы по имени параметра вместо позиции —
`=`, не `:` (та же нотация, что у записей `соответствие(ключ = значение)`):

```panos
функ вычесть(a: Число, b: Число) -> Число
    a - b
конец

функ старт() -> Число
    вычесть(b = 3, a = 10)
конец
```

Работает везде, где есть параметры/поля — обычные функции, конструкторы
структур (`Точка(x = 1, y = 2)`), методы (`объект.метод(n = 5)`).
Позиционную и именованную форму нельзя смешивать в одном вызове — либо
все аргументы по позиции, либо все по имени; все параметры при этом
всё равно обязательны (default-значений в panos нет), именование
только позволяет не помнить порядок объявления. Для интерфейсных
вызовов (переменная типа интерфейса) именованные аргументы пока не
поддержаны — только конкретные типы получателя.

## Явный `возврат`

```panos
функ проверить(x: Число) -> Строка
    если x < 0 тогда
        возврат "отрицательное"
    конец
    "неотрицательное"
конец

функ старт() -> Строка
    проверить(-5)
конец
```

`возврат` — обычный ранний выход, не обязателен, если функция и так
заканчивается нужным значением.

## Параметры неизменяемы

Параметры функций и лямбд **immutable по умолчанию** (как в Kotlin/
Swift) — переприсвоить их внутри тела нельзя:

```panos
функ f(x: Число) -> Число
    x = x + 1  // Type Error: попытка переприсвоить константу 'x'
    x
конец
```

Opt-out'а нет — если параметр нужно менять, копируйте его в локальный
`пер` в начале тела:

```panos
функ f(x: Число) -> Число
    пер счётчик = x
    счётчик = счётчик + 1
    счётчик
конец
```

Это ограничение касается только САМОГО биндинга параметра (нельзя
писать `x = ...`), не того, на что он ссылается — если параметр
структурного типа, его поля (`x.поле = ...`) по-прежнему мутируемы (у
panos структур reference semantics, см. [«Переменные»](basic-types.md#переменные)
и [«Структуры и методы»](structs-and-methods.md)).

## Точка входа

Программа начинает выполнение с функции `старт`. У неё может быть любой
подходящий возвращаемый тип — движок печатает результат `старт()`, если он
есть (именно так на этой странице печатается вывод под каждым примером).

## Функции как значения и лямбды

Тип функции записывается как `функ(Тип1, Тип2, ...) -> ТипРезультата`.
Функции — значения первого класса: их можно передавать как аргументы,
хранить в переменных.

```panos
функ применить(ф: функ(Число) -> Число, значение: Число) -> Число
    ф(значение)
конец

функ старт() -> Число
    применить(функ(x)
        x + 1
    конец, 41)
конец
```

Лямбда — это `функ(параметры) тело конец` без имени. Типы параметров и
результата лямбды можно не указывать явно, если type checker способен
вывести их из ожидаемого типа (выше — из типа параметра `ф` у `применить`).
Если ожидаемый тип функции неизвестен компилятору в месте, где стоит
лямбда, вывод типов параметров может не сработать — тогда нужно
аннотировать их явно, как у обычной функции (`функ(x: Число) ... конец`).

## Замыкания

Лямбда может ссылаться на переменные из ОКРУЖАЮЩЕЙ функции — панос
захватывает их автоматически, без специального синтаксиса:

```panos
функ создать_счётчик(старт: Число) -> функ() -> Число
    функ() -> Число
        старт
    конец
конец

функ старт() -> Число
    пер получить = создать_счётчик(10.0)
    получить()
конец
```

Захват — **по значению**: лямбда получает КОПИЮ значения переменной в
момент своего создания (не общую с внешней функцией ячейку). Более
поздние мутации переменной снаружи не видны внутри уже созданной
лямбды:

```panos
функ старт() -> Число
    пер n = 5.0
    пер л = функ() -> Число
        n
    конец
    n = 999.0
    л() // 5.0, а не 999.0 — снапшот на момент создания л
конец
```

По той же причине присвоить захваченной переменной ВНУТРИ лямбды нельзя
— это была бы мутация копии, ничего не меняющая снаружи, поэтому
компилятор репортит `Type Error` вместо тихого no-op:

```panos
функ старт() -> Число
    пер n = 5.0
    пер л = функ() -> Число
        n = 10.0 // Type Error: захваченная переменная 'n' неизменяема внутри лямбды
        n
    конец
    л()
конец
```

Захват работает и для вложенных лямбд (лямбда внутри лямбды) — символ,
объявленный вне обеих, доступен на любом уровне вложенности. При
[отправке замыкания другому процессу](processes.md) захваченные значения
копируются глубоко, тем же механизмом, что и остальные сообщения —
никакого общего мутируемого состояния между процессами замыкания не
создают.

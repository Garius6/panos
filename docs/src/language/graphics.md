# Графика (FFI фаза A)

> Всё на этой странице требует `импорт графика` и реальное окно — **недоступно**
> в браузерной песочнице (тот же принцип, что `фс`/`ос`/`сеть` — см.
> [«Модули» → ограничение в браузере](./modules.md#ограничение-в-браузере)).
> Примеры не запускаются на странице, запускайте локально: `./panos файл.ps`.

Модуль `графика` — первый FFI-шаг panos: статические Odin-биндинги
поверх [`vendor:raylib`](https://www.raylib.com/) (готовый биндинг из
поставки самого Odin-тулчейна, не своя `foreign import`-обвязка) —
доказательство, что panos может звать внешнюю C-библиотеку и рисовать
окно. Это **не** настоящий динамический FFI (`внешний`-декларации,
рантайм-линковка произвольных `.so`/`.dylib`) — та задача существенно
крупнее и запланирована отдельно.

Требует установленный raylib (`brew install raylib` на macOS, системные
пакеты `libraylib-dev` на Linux) — модуль линкуется статически на этапе
сборки `panos`, не рантайм-загрузкой.

## Точка (Vector2) и Цвет (Color)

У raylib'овых `Vector2`/`Color` простой memory layout (`[2]f32`/`[4]u8`)
— на panos-стороне это ОБЫЧНЫЕ тупы, без отдельного типа:

- точка — `(Число, Число)` (x, y)
- цвет — `(Число, Число, Число, Число)` (r, g, b, a — 0.0-255.0)

## Окно и цикл отрисовки

```panos-cli
импорт графика

функ старт() -> Пусто
    графика.инициализировать_окно(800, 450, "моё окно")
    графика.задать_fps(60)

    пока графика.окно_должно_закрыться() == ложь цикл
        графика.начать_рисование()
        графика.очистить_фон((20.0, 20.0, 20.0, 255.0))
        графика.нарисовать_прямоугольник((100.0, 100.0), (50.0, 30.0), (255.0, 0.0, 0.0, 255.0))
        графика.нарисовать_круг((300.0, 200.0), 25.0, (0.0, 255.0, 0.0, 255.0))
        графика.закончить_рисование()
    конец

    графика.закрыть_окно()
конец
```

`инициализировать_окно(ширина, высота: Целое, заголовок: Строка)` /
`закрыть_окно()` — открыть/закрыть окно один раз. `окно_должно_
закрыться() -> Булево` — `истина`, когда пользователь закрыл окно (крестик
или Esc) — стандартное условие выхода из игрового цикла. `начать_
рисование()`/`закончить_рисование()` окружают каждый кадр — всё
рисование (`нарисовать_*`) действует только между ними. `задать_fps(fps:
Целое)` — ограничение кадров в секунду (без него цикл рисует настолько
быстро, насколько может процессор).

## Рисование

- `очистить_фон(цвет)` — залить весь кадр цветом (обычно первым делом в
  кадре).
- `нарисовать_прямоугольник(позиция, размер, цвет)` — прямоугольник
  `размер.0 × размер.1`, левый верхний угол в `позиция`.
- `нарисовать_круг(центр, радиус: Число, цвет)`.

## Клавиатура и время кадра

```panos-cli
импорт графика

функ старт() -> Пусто
    графика.инициализировать_окно(400, 300, "клавиши")
    пока графика.окно_должно_закрыться() == ложь цикл
        пер дт = графика.время_кадра()
        если графика.клавиша_нажата(графика.клавиша_вверх()) тогда
            // дт (Число) — секунды с прошлого кадра, для скорости,
            // независимой от FPS: смещение = скорость * дт
        конец
        графика.начать_рисование()
        графика.очистить_фон((0.0, 0.0, 0.0, 255.0))
        графика.закончить_рисование()
    конец
    графика.закрыть_окно()
конец
```

`клавиша_нажата(код: Целое) -> Булево` принимает числовой код клавиши —
raylib'овские коды не доступны в panos как символьные константы напрямую
(нет механизма импорта чужих enum'ов), поэтому модуль экспортирует
геттеры на самые частые: `клавиша_вверх()`/`клавиша_вниз()`/
`клавиша_влево()`/`клавиша_вправо()`/`клавиша_пробел()`/`клавиша_
escape()` — каждый `() -> Целое`. `время_кадра() -> Число` — секунды,
прошедшие с предыдущего кадра (`dt`), для физики/движения, не зависящего
от FPS.

## Ограничение: графический цикл голодит остальные процессы

panos-планировщик (см. [«Процессы»](./processes.md)) кооперативный, без
преемптивности, один OS-поток на весь VM — переключение между процессами
происходит ТОЛЬКО когда текущий приостанавливается на пустом `получить()`/
`получить_сигнал()` или завершается/падает. `графика.*` — обычные
синхронные builtin'ы (как `фс.записать`), они НЕ приостанавливают
процесс. Значит `пока графика.окно_должно_закрыться() == ложь цикл ...
конец`, если внутри ни разу не встречается `получить()`/`получить_
сигнал()` с пустой очередью, выполняется одним огромным блоком — ВСЕ
остальные `запусти`-процессы (супервизоры, воркеры) не получат ни
единого такта планировщика, пока окно не закроется.

Сейчас в panos нет способа "уступить без ожидания сообщения" — совместить
графический цикл с ПАРАЛЛЕЛЬНО работающими акторами в одной программе
без явного протокола невозможно. `demos/pong.ps` этой проблемы не
показывает — не использует процессы, `старт()` синхронно ведёт весь
игровой цикл.

## Демо: pong

[`demos/pong.ps`](https://github.com/Garius6/panos/blob/main/demos/pong.ps)
— полноценный pong: весло игрока (стрелки вверх/вниз), простой ИИ-соперник,
отскок мяча от стен и вёсел, счёт печатается в консоль при пропущенном
мяче. Запуск: `./panos demos/pong.ps`.

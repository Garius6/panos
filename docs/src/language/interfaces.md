# Интерфейсы

```panos
тип Атакующий = интерфейс
    функ атаковать(урон: Число) -> Число
конец

тип Игрок = структура
    здоровье: Число
конец

реализация Атакующий для Игрок
    функ атаковать(это: Игрок, урон: Число) -> Число
        это.здоровье - урон
    конец
конец

функ старт() -> Число
    пер герой = Игрок(100)
    пер атакующий: Атакующий = герой
    атакующий.атаковать(15)
конец
```

`реализация Интерфейс для Тип` — сигнатуры методов в интерфейсе не
включают получателя (он неявный, добавляется в реализации первым
параметром). Типизация номинальная, не структурная: тип реализует
интерфейс только через явный блок `реализация ... для ...`, случайное
совпадение сигнатур методов не считается.

Интерфейсный тип у переменной/параметра даёт динамическую диспетчеризацию
— какой конкретно метод вызовется, решается по фактическому значению в
рантайме:

```panos
тип Фигура = интерфейс
    функ площадь() -> Число
конец

тип Круг = структура
    радиус: Число
конец

реализация Фигура для Круг
    функ площадь(это: Круг) -> Число
        это.радиус * это.радиус * 314 / 100
    конец
конец

тип Квадрат = структура
    сторона: Число
конец

реализация Фигура для Квадрат
    функ площадь(это: Квадрат) -> Число
        это.сторона * это.сторона
    конец
конец

функ общая_площадь(a: Фигура, b: Фигура) -> Число
    a.площадь() + b.площадь()
конец

функ старт() -> Число
    общая_площадь(Круг(10), Квадрат(4))
конец
```

## Перечисления тоже реализуют интерфейсы

`реализация` работает для `перечисление` точно так же, как для `структура`
— методу просто нужно разобрать варианты через `выбор`:

```panos
тип Фигура = интерфейс
    функ площадь() -> Число
конец

тип ФигураАДТ = перечисление
    Круг(Число)
    Квадрат(Число)
конец

реализация Фигура для ФигураАДТ
    функ площадь(это: ФигураАДТ) -> Число
        выбор это
            Круг(р) -> р * р * 314 / 100
            Квадрат(с) -> с * с
        конец
    конец
конец

функ старт() -> Число
    пер ф: Фигура = ФигураАДТ.Круг(10)
    ф.площадь()
конец
```

Так реализованы, например, `Опция`/`Результат` — оба обычные
generic-перечисления прелюдии, при этом умеют `Сравниваемое`, `Равнозначное`
и остальные [интерфейсы прелюдии](./prelude-interfaces.md), если такая
`реализация` для них написана.

## Generic-интерфейсы

Параметр типа может принадлежать самому интерфейсу, а не только
реализующей его структуре/перечислению — `T` в квадратных скобках после
имени интерфейса:

```panos
тип Контейнер[T] = интерфейс
    функ получить_значение() -> T
конец

тип Обёртка = структура
    x: Число
конец

реализация Контейнер для Обёртка
    функ получить_значение(это: Обёртка) -> Число
        это.x
    конец
конец

функ старт() -> Число
    пер о = Обёртка(42)
    о.получить_значение()
конец
```

В `реализация Контейнер для Обёртка` тип-аргумент писать не нужно — `T`
выводится из конкретной сигнатуры метода в теле реализации (здесь —
`Число`, потому что `получить_значение` возвращает `это.x: Число`). Разные
реализации одного generic-интерфейса свободно фиксируют разные `T`.

Главный пример в языке — [`Итерируемое[T]`](./prelude-interfaces.md#итерируемоеt)
из прелюдии, который управляет тем, что `для x в значение цикл` умеет
обходить произвольные пользовательские типы, а не только `Массив`/
`Соответствие`.

## Интерфейс из другого модуля

`реализация Модуль.Интерфейс для Тип` — интерфейс, объявленный в
ИМПОРТИРОВАННОМ модуле, реализуется точно так же, как локальный, только
с квалификацией имени:

```panos
// mods/фигуры.ps
экспорт тип Фигура = интерфейс
    функ площадь() -> Число
конец
```

```panos-cli
импорт "mods/фигуры" как фигуры

тип Круг = структура
    радиус: Число
конец

реализация фигуры.Фигура для Круг
    функ площадь(это: Круг) -> Число
        это.радиус * это.радиус * 314 / 100
    конец
конец

функ старт() -> Число
    пер ф: фигуры.Фигура = Круг(10)
    ф.площадь()
конец
```

Квалификация допустима ТОЛЬКО у имени интерфейса, в форме `реализация
Модуль.Интерфейс для Тип` — целевой тип (`Круг` выше) всегда объявляется
в том же файле, что и `реализация`. `реализация Модуль.Тип` (без `для`
— расширение методом чужого типа) не поддержано, синтаксическая ошибка.

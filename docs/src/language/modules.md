# Модули

## `импорт`/`экспорт`

```panos
импорт математика

функ старт() -> Число
    математика.степень(2, 10)
конец
```

`импорт имя_модуля` — либо встроенный модуль стандартной библиотеки
(`ввод_вывод`, `фс`, `ос`, `сеть`, `строки`, `время`), либо файл
`имя_модуля.ps` рядом (или в `модули/`, или по `PANOS_STDLIB`, см.
[«Использование CLI» → поиск модулей](../getting-started/cli.md#поиск-модулей)).
`математика`, `коллекции`, `флаги` и т.п. — НЕ встроенные модули, а обычные
`.ps`-файлы из `std/` (см. [«Стандартная библиотека»](./standard-library.md)),
резолвятся тем же файловым путём, что и пользовательские модули.
Объявление, помеченное `экспорт`, видно из импортирующего модуля через
`имя_модуля.Имя`:

```panos-cli
// файл геометрия.ps
экспорт тип Точка = структура
    x: Число
    y: Число
конец

экспорт функ сложить(a: Точка, b: Точка) -> Точка
    Точка(a.x + b.x, a.y + b.y)
конец
```

```panos-cli
// файл, который импортирует геометрию.ps
импорт геометрия

функ старт() -> Число
    пер p = геометрия.сложить(геометрия.Точка(1, 2), геометрия.Точка(3, 4))
    p.x + p.y
конец
```

Импорт не сливает имена в текущий scope — всегда через префикс
`модуль.Имя`, без `как` для сокращения (кроме `импорт X как Y`,
переименовывающего сам префикс).

## Прелюдия

`Опция`/`Результат` — исключение из правила «нужен импорт»: это обычные
generic-перечисления, объявленные во встроенной прелюдии, доступной
каждому файлу неявно, без единого `импорт`. Подробнее — в главе
[«Опция и Результат»](./option-and-result.md).

## Ограничение в браузере

Весь `std/*.ps` (`математика`, `коллекции`, `флаги`, `тест`,
`кодирование/json`, `кодирование/toml`, `сеть/http`) вшит прямо в
WASM-сборку демо (нет файловой системы, которой можно было бы его
отдать), так что импорт этих модулей работает и в интерактивном
редакторе. `строки` — тоже работает (это встроенный модуль языка, не
файл). Но у `флаги` и `сеть/http` это работает только частично — сами
типы/чистые функции доступны, а вызов того, что внутри них трогает `ос`
(`.распарсить()`) или `сеть` (`http.запросить` и т.п.), всё равно упадёт в
панику ниже — вшитость файла не убирает его внутренние зависимости.

Не работает то, что реально требует файловой системы/сети/стдина:
`фс`, `ос`, `ввод_вывод`, `сеть` — модули импортируются (не ошибка на
`импорт`), но вызов их функций падает в понятную панику "недоступно в
браузере" во время выполнения. Не работают и **локальные файловые
импорты** пользовательских модулей (пример с `геометрия.ps` ниже) — у
браузерного редактора всего один текстовый буфер, положить туда второй
файл негде.

Примеры с такими зависимостями на этой странице помечены как нерабочие
в интерактивном редакторе (нет кнопки «▶ Запустить»). Запустите их
локально: `./panos файл.ps`.

# Модули

## `импорт`/`экспорт`

```panos-cli
импорт математика

функ старт() -> Число
    математика.степень(2, 10)
конец
```

`импорт имя_модуля` — либо встроенный модуль стандартной библиотеки
(`ввод_вывод`, `фс`, `сеть`, `строки`, `математика`), либо файл `имя_модуля.ps`
рядом (или в `модули/`, или по `PANOS_STDLIB`, см.
[установку](../getting-started/installation.md)). Объявление, помеченное
`экспорт`, видно из импортирующего модуля через `имя_модуля.Имя`:

```panos-cli
// файл геометрия.ps
экспорт тип Точка = структура
    x: Число
    y: Число
конец

экспорт функ сложить(a: Точка, b: Точка) -> Точка
    Точка(a.x + b.x, a.y + b.y)
конец
```

```panos-cli
// файл, который импортирует геометрию.ps
импорт геометрия

функ старт() -> Число
    пер p = геометрия.сложить(геометрия.Точка(1, 2), геометрия.Точка(3, 4))
    p.x + p.y
конец
```

Импорт не сливает имена в текущий scope — всегда через префикс
`модуль.Имя`, без `как` для сокращения (кроме `импорт X как Y`,
переименовывающего сам префикс).

## Прелюдия

`Опция`/`Результат` — исключение из правила «нужен импорт»: это обычные
generic-перечисления, объявленные во встроенной прелюдии, доступной
каждому файлу неявно, без единого `импорт`. Подробнее — в главе
[«Опция и Результат»](./option-and-result.md).

## Ограничение в браузере

Примеры на этой странице помечены как нерабочие в интерактивном
редакторе (нет кнопки «▶ Запустить») — файловый импорт модулей требует
файловой системы, которой у браузера нет. Значит недоступны все модули,
которые сами требуют импорта (`ввод_вывод`, `фс`, `сеть`, `строки`,
`математика`). Работает только то, что встроено в язык напрямую:
арифметика, структуры/интерфейсы/дженерики/ADT, `Опция`/`Результат`,
методы массивов/карт — то есть практически весь язык, кроме
непосредственно стандартной библиотеки. Запустите такие примеры локально:
`./panos файл.ps`.

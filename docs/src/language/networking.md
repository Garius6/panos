# Сеть и HTTP

> Требует `импорт сеть` и реальную сеть — **недоступно** в браузерной
> песочнице (см. [«Модули» → ограничение в браузере](./modules.md#ограничение-в-браузере)).
> Примеры не запускаются на странице, запускайте локально: `./panos файл.ps`.

## `сеть`: TCP-клиент

```panos-cli
импорт сеть

функ старт() -> Строка
    пер соединение = сеть.подключиться("example.com", 80).ожидать("не удалось подключиться")
    соединение.отправить("GET / HTTP/1.1\r\nHost: example.com\r\nConnection: close\r\n\r\n")
    пер ответ = соединение.получить().получить("(пусто)")
    соединение.закрыть()
    ответ
конец
```

- `сеть.подключиться(хост, порт) -> Результат(Соединение, Ошибка)` — TCP,
  без TLS (`https://` не поддержан, см. ниже)
- `.отправить(текст) -> Результат(Число, Ошибка)`
- `.получить() -> Результат(Строка, Ошибка)` — читает до закрытия
  соединения удалённой стороной (до EOF)
- `.получить_строку() -> Результат(Строка, Ошибка)` — одна строка
- `.закрыть() -> Пусто`
- `сеть.кодировать_url(текст) -> Строка` — percent-encoding
  (`сеть.кодировать_url("а б")` → `"%D0%B0%20%D0%B1"`)

## `сеть/http`: HTTP(S)-клиент

Тонкая обёртка (`std/сеть/http.ps`) над builtin `сеть.http_запрос` —
настоящий HTTP-клиент (`external/odin-http` + OpenSSL для `https://`,
см. `core/vm_http_native.odin`), а не ручной сокет-парсер. Импортируется
по пути в кавычках, потому что имя содержит `/`:

```panos-cli
импорт "сеть/http" как http
импорт строки

функ старт() -> Строка
    пер ответ = http.запросить("https://example.com/").ожидать("запрос не удался")
    если ответ.успех() тогда
        ответ.тело
    иначе
        "код ответа: " + строки.из_числа(ответ.статус)
    конец
конец
```

И `http://`, и `https://`. Корректно обрабатывает `Content-Length` и
`Transfer-Encoding: chunked` (в отличие от старой реализации — та читала
тело до закрытия соединения, что портило chunked-ответы). Заголовки
ответа приходят в нижнем регистре (так их хранит native-клиент) —
`.заголовок(имя)` ищет без учёта регистра, прямой доступ через
`.заголовки.получить(...)` — с учётом.

Единственная системная зависимость во всём проекте (не вендорится, в
отличие от libffi) — на macOS/Linux нужен установленный OpenSSL 3.x
(`brew install openssl@3` / `apt install libssl-dev`) для сборки самого
`panos`; на Windows статические `.lib` уже в репозитории. Недоступно в
WASM-демо (тот же принцип, что `фс`/`сеть` — см. выше).

**Функции:**

- `http.запросить(ссылка) -> Результат(Ответ, Ошибка)` — GET
- `http.получить_с_заголовками(ссылка, заголовки) -> Результат(Ответ, Ошибка)`
- `http.запросить_head(ссылка) -> Результат(Ответ, Ошибка)` — HEAD
- `http.отправить_пост(ссылка, тело)` / `.отправить_put(ссылка, тело)` /
  `.отправить_patch(ссылка, тело) -> Результат(Ответ, Ошибка)` — и
  `_с_заголовками`-варианты каждого (доп. `Соответствие(Строка, Строка)`
  последним аргументом)
- `http.удалить(ссылка)` / `.удалить_с_заголовками(ссылка, заголовки) -> Результат(Ответ, Ошибка)` — DELETE
- `http.отправить_json(ссылка, значение: json.Значение) -> Результат(Ответ, Ошибка)`
  — сериализует через `кодирование/json`, ставит
  `Content-Type: application/json`
- `http.запросить_с_редиректами(ссылка, макс_переходов) -> Результат(Ответ, Ошибка)`
  — следует за `3xx`/`Location` (абсолютным и относительным), падает
  `Ошибка`, если лимит превышен
- `http.построить_query(параметры) -> Строка` — собирает
  `?ключ=значение&...` из `Соответствие(Строка, Строка)`, значения уже
  percent-encoded
- `http.разобрать_адрес(текст) -> Результат(Адрес, Ошибка)` — оставлено
  для обратной совместимости (только `http://`); сам `http_запрос` URL
  парсит внутри native-клиента, эта функция для запросов не нужна

**Тип `Ответ`:**

```panos
тип Ответ = структура
    статус:    Число
    заголовки: Соответствие(Строка, Строка)
    тело:      Строка
конец
```

Методы: `.успех()` (2xx), `.редирект()` (3xx), `.ошибка_клиента()` (4xx),
`.ошибка_сервера()` (5xx), `.заголовок(имя) -> Опция(Строка)` (без учёта
регистра).

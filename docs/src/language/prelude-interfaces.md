# Интерфейсы прелюдии

Шесть интерфейсов из прелюдии — она доступна каждому файлу без `импорт`,
как `Опция`/`Результат` (см. [«Модули»](./modules.md#прелюдия)) — дают
пользовательским типам операторный синтаксис (`<`, `==`, `+` и т.д.) и
управляют тем, как значение печатается, копируется и обходится в `для-in`.
Реализуются они так же, как любой другой интерфейс — `реализация Интерфейс
для Тип` — просто typechecker и компилятор узнают эти конкретные имена и
превращают вызов оператора/встроенной функции в вызов нужного метода.

## Сравниваемое

```panos
тип Деньги = структура
    копейки: Число
конец

реализация Сравниваемое для Деньги
    функ сравнить(это: Деньги, другое: Деньги) -> Число
        это.копейки - другое.копейки
    конец
конец

функ старт() -> Булево
    Деньги(150) < Деньги(300)
конец
```

`сравнить` возвращает `Число` по конвенции отрицательное/ноль/
положительное — этого достаточно, чтобы компилятор выразил через него все
четыре оператора: `<`, `>`, `<=`, `>=`. У численных типов `<`/`>` и так
работают из коробки; для остальных (структур, перечислений) реализация
`Сравниваемое` — единственный способ получить эти операторы, дефолта нет.

## Равнозначное

```panos
тип Пользователь = структура
    id:  Число
    имя: Строка
конец

реализация Равнозначное для Пользователь
    функ равно(это: Пользователь, другой: Пользователь) -> Булево
        это.id == другой.id
    конец
конец

функ старт() -> Булево
    Пользователь(1, "Аня") == Пользователь(1, "Дубликат")
конец
```

В отличие от `Сравниваемое`, это **опционально**: `==`/`<>` и так работают
на любых типах — по умолчанию сравниваются все поля структурно, рекурсивно.
`реализация Равнозначное` переопределяет это поведение для типов, которым
дефолт не подходит (как выше — сравнение только по `id`, без учёта имени).

## Арифметика: Складываемое, Вычитаемое, Умножаемое, Делимое

```panos
тип Вектор = структура
    x: Число
    y: Число
конец

реализация Складываемое для Вектор
    функ сложить(это: Вектор, другой: Вектор) -> Вектор
        Вектор(это.x + другой.x, это.y + другой.y)
    конец
конец

функ старт() -> Число
    пер сумма = Вектор(1, 2) + Вектор(3, 4)
    сумма.x + сумма.y
конец
```

Четыре независимых интерфейса — `Складываемое`/`Вычитаемое`/`Умножаемое`/
`Делимое`, методы `сложить`/`вычесть`/`умножить`/`разделить` — вместо
одного, потому что операции независимы: вектор естественно поддерживает
сложение и вычитание, но не деление на другой вектор. Реализуйте только те,
которые осмысленны для вашего типа.

## Печатаемое

```panos
тип Цвет = перечисление
    Красный
    Зелёный
    Синий
конец

реализация Печатаемое для Цвет
    функ вСтроку(это: Цвет) -> Строка
        выбор это
            Красный -> "красный"
            Зелёный -> "зелёный"
            Синий -> "синий"
        конец
    конец
конец

функ старт() -> Строка
    Цвет.Красный.вСтроку()
конец
```

Опционально — без реализации `ввод_вывод.печать`/`.строка` печатают
structural-дамп полей (`Тип(поле1, поле2, ...)`). Реализуйте `Печатаемое`,
чтобы задать собственное текстовое представление; оно применяется
автоматически везде, где значение превращается в текст этим путём.

## Копируемое

```panos
тип Узел = структура
    значение: Число
конец

реализация Копируемое для Узел
    функ клонировать(это: Узел) -> Узел
        Узел(это.значение)
    конец
конец

функ старт() -> Число
    пер a = Узел(1)
    пер b = a.клонировать()
    b.значение = 99
    a.значение
конец
```

Структуры в panos — reference-типы: присваивание (`пер b = a`) копирует
указатель, не поля, так что мутация через `b` была бы видна и через `a`.
`.клонировать()` — единственный способ получить настоящую независимую
копию. Тело пишется руками (нет auto-derive), и для **глубокой** копии
обязано само рекурсивно звать `.клонировать()` на вложенных
struct-полях — иначе получится поверхностная копия с расшаренными
вложенными объектами. Тот же метод используется как copy-on-send override
для [сообщений между процессами](./processes.md#copy-on-send).

## Итерируемое[T]

```panos
тип СчётчикДо = структура
    текущее: Число
    предел: Число
конец

реализация Итерируемое для СчётчикДо
    функ следующий(это: СчётчикДо) -> Опция(Число)
        если это.текущее >= это.предел тогда
            Опция.Нет()
        иначе
            это.текущее = это.текущее + 1
            Опция.Есть(это.текущее)
        конец
    конец
конец

функ старт() -> Число
    пер сч = СчётчикДо(0, 5)
    пер сумма = 0
    для x в сч цикл
        сумма = сумма + x
    конец
    сумма
конец
```

`Итерируемое[T]` — единственный в языке *generic*-интерфейс (сам параметр
`T` объявлен у интерфейса, не только у реализующего типа — см.
[«Интерфейсы» → generic-интерфейсы](./interfaces.md#generic-интерфейсы)).
Единственный метод `следующий() -> Опция(T)` возвращает `Опция.Есть(x)` для
очередного элемента и `Опция.Нет()`, когда обход закончен. Реализующий тип
сам хранит и мутирует своё состояние между вызовами — `это` неизменяемый
биндинг, но поля через `это.поле = ...` мутируемы (reference semantics
структур). `для x в значение цикл ... конец` на типе с `реализация
Итерируемое` вызывает `.следующий()` в цикле автоматически; без реализации
`для-in` работает только на `Массив`/`Соответствие` напрямую.

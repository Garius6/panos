# Структуры и методы

## Объявление и конструктор

```panos
тип Точка = структура
    x: Число
    y: Число
конец

функ старт() -> Число
    пер p = Точка(10, 20)
    p.x + p.y
конец
```

Конструктор — вызов по имени типа, аргументы в порядке объявления полей.
Поле-тупл, поле-структура и т.п. — обычные типы, ничего особого:

```panos
тип Игрок = структура
    имя: Строка
    координаты: (Число, Число)
конец

функ старт() -> Число
    пер игрок = Игрок("Гайдар", (1, 2))
    игрок.координаты.0
конец
```

## Методы

Объявляются в блоке `реализация Тип`. Первый параметр метода — получатель,
его тип должен совпадать с типом владельца; имя параметра — обычное имя,
никакого зарезервированного `self`/`this`, идиоматично называть `это`:

```panos
тип Счётчик = структура
    значение: Число
конец

реализация Счётчик
    функ увеличить(это: Счётчик, шаг: Число) -> Число
        это.значение + шаг
    конец
конец

функ старт() -> Число
    пер c = Счётчик(10)
    c.увеличить(5)
конец
```

Структуры неизменяемы через методы напрямую — `увеличить` выше не меняет
`c`, а возвращает новое значение. Присваивание полю целиком переопределяет
переменную:

```panos
тип Счётчик = структура
    значение: Число
конец

реализация Счётчик
    функ увеличить(это: Счётчик, шаг: Число) -> Счётчик
        Счётчик(это.значение + шаг)
    конец
конец

функ старт() -> Число
    пер c = Счётчик(10)
    c = c.увеличить(5)
    c.значение
конец
```

## Деструктуризация

`пер Тип(a, b, ...) = значение` разбирает структуру на отдельные
биндинги по полям — в том же порядке, в каком поля объявлены (симметрично
конструктору выше):

```panos
тип Точка = структура
    x: Число
    y: Число
конец

функ старт() -> Число
    пер точка = Точка(3, 4)
    пер Точка(x, y) = точка
    x + y
конец
```

Тот же синтаксис без имени типа — `пер (a, b, ...) = кортеж` — разбирает
тупл:

```panos
функ старт() -> Число
    пер (a, b, c) = (1, 2, 3)
    a + b + c
конец
```

Обе формы требуют точного совпадения количества имён с количеством
полей/элементов — несовпадение или неверный тип структуры (`пер
Вектор(x, y) = точка`, где `точка` на самом деле `Точка`) — ошибка
компиляции. `конст (a, b) = ...` тоже работает — все извлечённые имена
становятся неизменяемыми, как обычный [`конст`](./basic-types.md#переменные).

### Именованная деструктуризация

У структурной формы (не у тупла) есть именованный вариант — `пер
Тип(поле: имя, ...) = значение` — извлекает поля по имени, а не по
позиции. Порядок в списке не обязан совпадать с порядком объявления
полей, и можно извлечь только часть полей:

```panos
тип Точка = структура
    x: Число
    y: Число
    z: Число
конец

функ старт() -> Число
    пер точка = Точка(1, 2, 3)
    пер Точка(y: b, x: a) = точка
    пер Точка(z: c) = точка
    a * 100 + b * 10 + c
конец
```

Смешивать именованную и позиционную форму в одной деструктуризации
нельзя — либо все поля по имени, либо все по позиции. Неизвестное имя
поля или повтор одного и того же поля — ошибка компиляции. У тупла
именованной формы нет — только позиционная.

# Литература по теории систем типов (на русском)

Список составлен по темам, которые всплывали при обсуждении type checker'а
панос: control-flow narrowing, CFG/dataflow для dead code, constraint-based
inference, generics. Порядок внутри раздела — от входного к продвинутому.

## Формальная логика (natural deduction, sequent calculus)

- Верещагин Н.К., Шень А. — *«Лекции по математической логике и теории
  алгоритмов»* (Ч.1 «Начала теории множеств», Ч.2 «Языки и исчисления»).
  МЦНМО, свободно доступна. Лучший вход — компактно, строго, с упражнениями.
- Мендельсон Э. — *«Введение в математическую логику»* (перевод).
  Стандартный университетский учебник.
- Непейвода Н.Н. — *«Прикладная логика»*. Написана по-русски, логика в
  привязке к программированию.

## Индукция по структуре

- Верещагин, Шень — та же серия, раздел про индукцию.
- Виленкин Н.Я. — *«Индукция. Комбинаторика»*. Числовая индукция, не
  structural — разогрев, не основной источник.

## Теория множеств

- Верещагин, Шень — *«Начала теории множеств»* (Ч.1 той же серии).
- Халмош П. — *«Наивная теория множеств»* (перевод).

## Lambda calculus / просто типизированное λ→ / HM / subtyping

- Пирс Б. — *«Типы в языках программирования»* (Types and Programming
  Languages / TAPL). Изд. «Добросвет»/КДУ, официальный перевод.
  **Главная книга** — покрывает ровно нужный стек: λ-calculus,
  unification-based inference, subtyping, System F.

## Unification algorithm (Robinson)

- Клоксин У., Меллиш К. — *«Программирование на языке Пролог»* (перевод).
  Алгоритм унификации разобран подробно — тот же алгоритм, что в
  `unify_types`, другой домен применения.
- В TAPL (Пирс) тоже есть, но конспективно.

## Dataflow-анализ, lattices, fixpoint (Knaster-Tarski)

Под CFG/dead-code-with-loops.

- Ахо А., Ульман Д., Сети Р., Лам М. — *«Компиляторы: принципы, технологии
  и инструментарий»* (Dragon Book). Изд. «Вильямс», полный перевод. Глава
  про dataflow analysis — worklist, lattice, fixpoint. **Обязательна**,
  если строить CFG-pass.

## Order theory / решётки (опционально, фундамент под dataflow)

- Гретцер Г. — *«Общая теория решёток»* (перевод, изд. «Мир»). Продвинутый
  уровень.

## Теория графов (базовая, под CFG/reachability)

- Харари Ф. — *«Теория графов»* (перевод). Стандарт.
- Оре О. — *«Теория графов»* (перевод). Чуть более элементарная подача.

## Абстрактная алгебра (опционально, под type classes/algebraic subtyping)

- Кострикин А.И. — *«Введение в алгебру»*. Написана по-русски, три тома.
- Ван дер Варден Б.Л. — *«Алгебра»* (перевод). Классика, сжато.

## Bidirectional typing / Algorithm W / type classes (из обсуждения
## constraint-based inference и generic-интерфейсов для панос)

Все источники ниже — статьи, не главы книг, перевода на русский нет ни у
одной. TAPL (выше) покрывает Algorithm W/HM конспективно — для уровня
panos, возможно, достаточно TAPL; к оригиналам — если нужна точная
формальная формулировка или сравнение подходов из первых рук.

- Dunfield J., Krishnaswami N. — *"Bidirectional Typing"* (ACM Computing
  Surveys, 2021). Систематизирующий survey всей области — то самое
  "bidirectional + HM", что обсуждали как направление для тайпчекера
  панос. Свободно доступна (arXiv). Лучший единый вход в тему.
- Milner R. — *"A Theory of Type Polymorphism in Programming"* (1978).
  Оригинальная работа, Algorithm W в исходной форме.
- Pierce B., Turner D. — *"Local Type Inference"* (TOPLAS, 2000).
  Оригинальная работа по bidirectional-стилю (check/synthesize) —
  предшественник survey выше, более узкий фокус.
- Wadler P., Blott S. — *"How to Make Ad-hoc Polymorphism Less Ad Hoc"*
  (POPL, 1989). Оригинальная статья про type classes — сравнение с
  interface-based диспетчеризацией (обсуждали разницу type class vs
  интерфейс) видно прямо из мотивации статьи.

## Теория категорий (опционально, не нужна для панос сейчас)

- Маклейн С. — *«Категории для работающего математика»* (перевод,
  «Физматлит»). Официальный, тяжёлый вход.
- Милевски Б. — *«Теория категорий для программистов»*. Неофициальный
  community-перевод *Category Theory for Programmers* — мягче вход для
  CS-читателя, чем Маклейн.

## Рекомендуемый порядок чтения

1. Пирс, TAPL.
2. Dragon Book, глава dataflow analysis.
3. Верещагин/Шень — при нехватке базы по логике/множествам.
4. Остальное — по мере упирания в конкретную стену (generics глубже →
   алгебра/категории, CFG сложнее → решётки/графы, диспетчеризация/
   bounded polymorphism глубже → раздел bidirectional/Algorithm W/type
   classes).

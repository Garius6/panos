# Contract: Грамматика синтаксиса ADT и `выбор`

Расширение существующей грамматики Panos. Точки, где парсер меняется —
`parse_top_level`, `parse_primary`. Разделитель `SEP` — `;` или перевод
строки (`consume_semicolon_or_newline`).

## 1. Объявление ADT

```
EnumDecl        := ("экспорт")? "тип" IDENT "=" "перечисление" VariantList "конец"
VariantList     := Variant (SEP Variant)* SEP?
Variant         := IDENT ("(" TypeList ")")?
TypeList        := Type ("," Type)*
```

Требования:

- `VariantList` не пуст; иначе ошибка парсера: «перечисление должно
  объявлять хотя бы один вариант».
- Скобки `()` без содержимого запрещены: `Точка` — без параметров,
  `Точка()` — ошибка парсера («либо без скобок, либо со списком типов»).
- Имена вариантов внутри одного `EnumDecl` уникальны.

Диспетчеризация в `parse_top_level`: при `.TypeDecl` смотрим
`peek_token[current_idx + 3].kind`; ветки — `.Struct` → структура,
`.Interface` → интерфейс, `.Enum` → перечисление (новое), иначе — ошибка
«ожидалась структура, интерфейс или перечисление».

## 2. Выражение `выбор`

```
MatchExpr := "выбор" Expr Arm (SEP Arm)* SEP? "конец"
Arm       := Pattern "->" StmtBlock
StmtBlock := Stmt (SEP Stmt)*
```

Требования:

- Хотя бы одна ветка; иначе ошибка парсера.
- `StmtBlock` в ветке — та же грамматика, что и тела `если`; последний
  Stmt-expr даёт значение ветки.
- Ветка с шаблоном `_` MUST быть последней в `выбор`; всё, что после неё —
  ошибка компиляции (Q4). Правило проверяется в type checker'е, а не в
  парсере, чтобы сообщение было согласовано со списком «недостижимые
  ветки».

`MatchExpr` — первичное выражение, интегрируется в `parse_primary` под
токеном `.Match`.

## 3. Шаблоны

```
Pattern        := "_"                             #wildcard
                | IDENT                           #ident-либо-конструктор без полей (см. R2)
                | IDENT "(" PatternList ")"        #конструктор с полями
                | IDENT "." IDENT ("(" PatternList ")")?   #квалификация типом
                | IDENT "." IDENT "." IDENT ("(" PatternList ")")?  #модуль.Тип.Вариант (Q3)
PatternList    := Pattern ("," Pattern)*
```

Правила разбора:

- Голый `IDENT` без скобок и без точки: парсер оставляет как
  `Pattern_Ident{name}`. Резольвер, увидев, что имя разрешается в
  `Symbol_Kind.Enum_Variant` и вариант не имеет полей, переклассифицирует
  его в `Pattern_Constructor{name = имя, args = {}}`. Иначе — биндер.
- `IDENT.IDENT` без скобок допускается только если сам вариант объявлен
  без полей.
- Не поддерживаются: литеральные шаблоны, or-шаблоны, охраны, tuple-
  шаблоны. Попытка их использовать даёт ошибку парсинга
  («такой шаблон пока не поддерживается»).

## 4. Позиция `выбор` в языке

`выбор` — выражение, но, как и `если`, может использоваться на месте
statement'а через `Expr_Stmt`. Тип — общий супертип тел веток
(вычисляется type checker'ом).

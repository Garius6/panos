# Quickstart: ADT и pattern-matching в Panos

Смоук-программа для ручной и автоматической проверки US1–US3. Файл может
быть сохранён под `specs/001-adt-pattern-matching/quickstart.ps` и
запущен командой `just debug-file specs/001-adt-pattern-matching/quickstart.ps`.

## Программа

```panos
// Объявление ADT (US1)
тип Фигура = перечисление
    Точка
    Круг(Число)
    Прямоугольник(Число, Число)
конец

// Функция-разбор через `выбор` (US2, US3)
функ площадь(ф: Фигура) -> Число
    возврат выбор ф
        Точка -> 0
        Круг(р) -> р * р * 314 / 100         // приближённо π
        Прямоугольник(ш, в) -> ш * в
    конец
конец

функ старт()
    // US1: печать самих ADT-значений (формат ИмяВарианта(...))
    печать(Точка)                             // Точка
    печать(Круг(3))                           // Круг(3)
    печать(Прямоугольник(4, 5))               // Прямоугольник(4, 5)

    // US2/US3: разбор через `выбор`
    печать(площадь(Точка))                    // 0
    печать(площадь(Круг(3)))                  // ~2826
    печать(площадь(Прямоугольник(4, 5)))      // 20
конец
```

## Ожидаемый вывод

```
Точка
Круг(3)
Прямоугольник(4, 5)
0
2826
20
```

## Что этот пример проверяет

| Сценарий | Отражение |
|----------|-----------|
| US1 AC-1 | Три конструктора (без полей, один, два) успешно создают значения ADT. |
| US1 AC-2 | Проверяется в отдельном e2e-тесте (не в этой программе) — заменяем `Круг(3)` на `Круг("три")` и ждём отказ проверки типов. |
| US2 AC-1 | `площадь` возвращает разный `Число` для каждого варианта. |
| US2 AC-2 | Биндеры `р`, `ш`, `в` видны внутри своих веток и типизированы как `Число`. |
| US3 AC-1 | Если убрать ветку `Точка`, программа отвергается type checker'ом со списком непокрытых. |

## Как это подключается к e2e_test.odin

Для US1..US3 в `e2e_test.odin` добавляются самостоятельные тестовые
функции с `run_code(строка_программы)`:

- `test_adt_declare_and_construct` — версия программы без `выбор`,
  проверяет только US1.
- `test_match_returns_expected` — версия выше, проверяет U2 и U3
  позитивные случаи.
- `test_match_missing_variant_rejected` — заранее известное сообщение
  ошибки при удалённой ветке.
- `test_option_result_match` (US4) — минимальная программа с
  `выбор` над `Опция(Число)` и `Результат(Строка, Ошибка)`.

Каждый тест независим и работает отдельно.

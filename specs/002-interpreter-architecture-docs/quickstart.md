# Quickstart: как проверить, что документация работает

Эта фича не запускается как программа — "quickstart" здесь означает: как
собрать документацию локально и как самостоятельно прогнать один из
Independent Test сценариев spec.md, чтобы убедиться, что раздел
"Архитектура интерпретатора" реально закрывает цель ("сам вношу изменения
без LLM").

## 1. Собрать mdBook локально

```sh
cd docs
mdbook build
```

Успех: команда завершается без ошибок, `docs/book/` содержит
`architecture/*.html`, все внутренние ссылки (`[текст](../lexer.md)` и
т.п.) резолвятся — `mdbook build` сам репортит битые ссылки как ошибку.

Локальный просмотр с live-reload при правке:

```sh
mdbook serve
```

## 2. Прогнать один Independent Test сценарий (User Story 1, P1)

Сценарий из spec.md: изолированная правка одной стадии pipeline, опираясь
ТОЛЬКО на текст документации.

1. Открыть `docs/src/architecture/parser.md` (без чтения `core/parser.odin`
   заранее).
2. Найти в разделе "Точки входа для типичной правки" запись про добавление
   нового бинарного оператора.
3. Убедиться, что документация называет ВСЕ файлы: `parser.odin`
   (`infix_bp`/`prefix_bp`), `type_cheker.odin` (`infer_binary_expr`),
   `compiler.odin`+`vm.odin` (opcode) — без пропущенных мест.
4. Успех: список полный (сверить с `core/parser.odin`/`type_cheker.odin`
   ПОСЛЕ, не до, самопроверки — если что-то не совпало, глава требует
   правки, а не читатель ошибся).

## 3. Прогнать сценарий для LSP (User Story 2, P2)

1. Открыть `docs/src/architecture/lsp.md`.
2. Найти подраздел "Известные ограничения".
3. Убедиться, что явно названы ОБЕ границы: (а) rename/references видят
   только открытые документы, не весь проект на диске; (б) `Symbol_Id` не
   сравним между графами разных документов, сравнение декларации — по
   (путь, byte-span).
4. Открыть `docs/src/architecture/known-pitfalls.md`, убедиться, что оба
   пункта (map[key]-сегфолт, json.marshal/указатели) присутствуют с полным
   набором полей (Симптом/Причина/Безопасный паттерн/Источник).

## 4. Быстрая проверка FR-012 (ссылки на код, не общие фразы)

Для ЛЮБОЙ главы: открыть файл, убедиться, что почти каждый абзац содержит
хотя бы один элемент `` `код` `` — имя файла, структуры или функции. Абзац
без единой такой отметки в разделах "Что"/"Зачем"/"Почему так" — нарушение
контракта (`contracts/chapter-template.md`), требует правки перед тем, как
считать главу готовой.

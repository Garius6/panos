# Contract: CLI-команды pan

**Feature**: `003-pan-package-manager`

Пакет собран и запускается как `panos <путь-к-start.ps-пакета-pan> <подкоманда>
[аргументы...]` (сам pan — обычная panos-программа). Ниже — контракт видимого
пользователю поведения каждой подкоманды: аргументы, побочные эффекты, exit
code, что печатается в stdout/stderr.

## `pan init`

- **Аргументы**: нет обязательных. Опционально — имя пакета (иначе берётся имя
  текущей директории).
- **Предусловие**: в текущей директории нет `pan.toml`.
- **Побочные эффекты**: создаёт `pan.toml` (имя, версия `0.1.0`, пустые
  зависимости) и `start.ps` (минимальный исходник) в текущей директории.
- **Exit code**: `0` при успехе; `1` если `pan.toml` уже существует (файл не
  перезаписывается).
- **stdout/stderr**: при ошибке — понятное сообщение в stderr, без создания
  файлов.

## `pan add <источник> [версия]`

- **Аргументы**: `<источник>` — git URL зависимости, обязателен. `[версия]` —
  semver-диапазон или точный коммит/ветка; при отсутствии — последняя
  доступная версия (последний семверный тег источника).
- **Предусловие**: текущая директория — внутри пакета (найден `pan.toml`
  upward-поиском).
- **Побочные эффекты (при успехе)**:
  1. Клонирует источник (или использует уже закэшированный коммит) в
     `~/.pan/cache/...`.
  2. Разрешает граф зависимостей (включая транзитивные из манифеста
     клонированного пакета).
  3. Записывает зависимость в `pan.toml[зависимости]` и полный граф в
     `pan.lock`.
  4. Раскладывает разрешённые пакеты в `модули/` корня пакета.
- **Побочные эффекты (при ошибке)**: `pan.toml`/`pan.lock`/`модули/` не
  изменяются (all-or-nothing для этой конкретной команды — в отличие от
  `pan install`, где частичный прогресс между разными зависимостями допустим,
  см. FR-010).
- **Ошибки**:
  - источник недоступен/не существует → exit `1`, понятное сообщение, ничего
    не изменено;
  - имя пакета-зависимости (canonical name) совпадает с уже добавленной
    зависимостью с другим источником → exit `1`, требует явного алиаса
    (сообщение подсказывает синтаксис алиаса), ничего не изменено;
  - конфликт версий/цикл при разрешении графа → exit `1`, сообщение называет
    конкретные конфликтующие пакеты и цепочки (SC-003), ничего не изменено.
- **Exit code**: `0` при успехе, `1` при любой из ошибок выше.

## `pan remove <имя>`

- **Аргументы**: `<имя>` — локальное имя зависимости (ключ в
  `pan.toml[зависимости]`), обязателен.
- **Побочные эффекты**: удаляет запись из `pan.toml` и `pan.lock`
  (пересчитывает граф без удалённой зависимости — транзитивные зависимости,
  на которые больше никто не ссылается, тоже уходят из lock и `модули/`).
- **Ошибки**: имя не найдено в манифесте → exit `1`, ничего не изменено.
- **Exit code**: `0` при успехе, `1` если имя не найдено.

## `pan install`

- **Аргументы**: нет.
- **Побочные эффекты**:
  - Если `pan.lock` соответствует `pan.toml` — раскладывает в `модули/` из
    локального кэша уже разрешённые версии; сеть не используется, если кэш
    полон (SC-004).
  - Если в `pan.toml` есть зависимости, отсутствующие в `pan.lock` — разрешает
    их, обновляет `pan.lock`, сообщает об этом явно в stdout.
  - Если зависимость уже успешно установлена в рамках этого запуска, а
    следующая упала с ошибкой — уже установленные остаются на диске (FR-010),
    сеть/git не откатываются.
- **Ошибки**: повреждённый/некорректный `pan.lock` → exit `1` до любых
  сетевых действий, с объяснением, что именно не так (FR-009); недоступность
  источника посреди установки нескольких зависимостей → exit `1`, уже
  установленное не удаляется, повторный `pan install` не должен заново
  скачивать то, что уже успешно легло.
- **Exit code**: `0` при полном успехе, `1` при любой ошибке (частичной или
  полной).

## `pan run [аргументы-программы...]`

- **Аргументы**: всё после `pan run` передаётся как program args в
  запускаемую программу (не интерпретируется самим pan).
- **Предусловие**: текущая директория — внутри пакета (upward-поиск
  `pan.toml`); все зависимости из манифеста присутствуют в `модули/`
  (сверяется с `pan.lock`, без обращения к сети).
- **Побочные эффекты**: спавнит `panos <точка_входа> [program args]` как
  дочерний процесс с рабочей директорией = найденный корень пакета (FR-013),
  используя новый builtin `ос.выполнить` (см.
  `contracts/stdlib-additions.md`).
- **Ошибки**: зависимость объявлена, но не установлена → exit `1` до попытки
  спавна `panos`, сообщение перечисляет недостающие зависимости (FR-014,
  вместо невнятной ошибки резолва импорта).
- **Exit code**: пробрасывается ровно тем же, что вернул дочерний процесс
  `panos`, кроме случая ошибки самого pan (недостающие зависимости) — тогда
  `1` и `panos` вообще не запускается.
- **stdout/stderr**: пробрасываются 1:1 из дочернего процесса (без
  дополнительного форматирования/буферизации со стороны pan).

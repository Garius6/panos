# Data Model: Pan — пакетный менеджер для panos

**Feature**: `003-pan-package-manager` | **Date**: 2026-07-21

Источник — раздел "Key Entities" в `spec.md`, уточнённый решениями из
`research.md`. Формат сериализации — TOML (`std/кодирование/toml.ps`) для двух
persisted-сущностей (Манифест, Lock-файл); остальные — исключительно
in-memory во время выполнения команд pan.

## Манифест пакета (`pan.toml`)

Persisted, человеко-редактируемый. Корень файла — соответствие с полями:

| Поле | Тип | Обязательность | Описание |
|------|-----|-----------------|----------|
| `имя` | Строка | обязательно | Canonical-имя пакета — то, под которым его импортируют потребители. |
| `версия` | Строка (semver) | обязательно | Версия пакета, по умолчанию `0.1.0` при `pan init`. |
| `точка_входа` | Строка (путь) | опционально, default `start.ps` | Файл, который запускает `pan run`. |
| `зависимости` | Соответствие(Строка → Зависимость) | опционально, default пусто | Ключ — локальное имя ссылки (обычно совпадает с canonical-именем, если нет коллизии/алиаса). |

**Валидация**:
- `имя` не пусто, не содержит `/` (namespaced-имена — только у зависимостей
  через путь импорта, не у собственного имени пакета).
- `версия` — валидный semver (`MAJOR.MINOR.PATCH`).
- Повторный `pan init` над директорией с уже существующим `pan.toml` —
  ошибка, файл не перезаписывается (FR-001, US1 acceptance #2).

## Lock-файл (`pan.lock`)

Persisted, генерируется и обновляется автоматически (FR-006, FR-009). Корень —
соответствие с полями:

| Поле | Тип | Описание |
|------|-----|----------|
| `версия_формата` | Число | Версия схемы lock-файла — для будущей миграции формата. |
| `пакеты` | Массив(Lock_Запись) | Полный разрешённый граф: и прямые, и транзитивные зависимости. |

**Lock_Запись**:

| Поле | Тип | Описание |
|------|-----|----------|
| `имя` | Строка | Canonical-имя пакета (совпадает с ключом в модуле `модули/`). |
| `источник` | Строка (git URL) | Откуда получен. |
| `разрешённая_версия` | Строка | Точный тег/коммит, к которому привязан. |
| `коммит` | Строка | Точный git-коммит (для верификации при повторном `pan install`). |
| `зависит_от` | Массив(Строка) | Имена пакетов, от которых зависит эта запись — граф для диагностики конфликтов (SC-003). |

**Валидация** (FR-009):
- Каждое имя из `зависит_от` должно существовать среди `пакеты[].имя`, иначе
  lock-файл повреждён — pan отказывается работать до исправления.
- Не должно быть двух записей с одинаковым `имя` (единая версия на граф,
  research.md п.1) — если найдено, lock-файл повреждён/рассинхронизирован.

## Зависимость (запись в `pan.toml[зависимости]`)

In-manifest структура (не отдельный файл):

| Поле | Тип | Описание |
|------|-----|----------|
| `источник` | Строка (git URL) | Адрес git-репозитория. |
| `версия` | Строка | Semver-диапазон (`^1.2.0`, `~1.2.0`, `>=1.0.0`) либо точный коммит/ветка (research.md п.2). |
| `алиас` | Строка, опционально | Задаётся только при коллизии canonical-имён двух разных источников (edge case в spec.md). |

## Разрешённая зависимость (in-memory, во время `pan add`/`pan install`)

Промежуточная сущность построения графа — не сериализуется отдельно, итог
разрешения записывается в Lock_Запись:

| Поле | Тип | Описание |
|------|-----|----------|
| `имя` | Строка | Canonical-имя (из манифеста самой зависимости после клонирования). |
| `источник` | Строка | Git URL. |
| `запрошенное_ограничение` | Строка | Как было указано у каждого из потребителей в графе (может быть несколько — для диагностики конфликта). |
| `кандидаты_версий` | Массив(Строка) | Доступные git-теги источника, из которых выбирается разрешение. |
| `выбранная_версия` | Строка \| Пусто | Итог разрешения; `Пусто`, если конфликт (FR-005). |

## Локальный кэш пакетов

Не структура данных в памяти pan, а конвенция на диске:

```
~/.pan/cache/<host>/<org>/<repo>/<commit>/...   # клонированные исходники зависимости на конкретном коммите
```

Наполняется через `git clone` + `git checkout <коммит>` (research.md п.5, п.8).
Один коммит — одна неизменяемая копия, переиспользуется всеми проектами и
всеми версийными ограничениями, разрешившимися в этот же коммит.

## Директория `модули/`

Конвенция на диске в корне пакета-потребителя:

```
<корень-пакета>/модули/<canonical-имя>.ps            # однофайловый пакет
<корень-пакета>/модули/<группа>/<canonical-имя>.ps   # namespaced-пакет (см. std/кодирование/*.ps как прецедент)
```

Наполняется копированием (не симлинком — переносимость на Windows, где
симлинки требуют повышенных прав) из локального кэша при `pan install`/`pan add`.
Резолвится компилятором panos "как есть" — `core/resolver_import_native.odin:12`,
без изменений в `core/`.

## Корень пакета (in-memory, вычисляется при каждом вызове pan)

Не persisted — результат upward-поиска `pan.toml` от текущей рабочей
директории (research.md п.7), используется как:
- база для пути `модули/` при `pan add`/`pan remove`/`pan install`;
- рабочая директория дочернего процесса `panos` при `pan run` (FR-013).

## State transitions

Единственная сущность с жизненным циклом — Lock_Запись/зависимость в графе:

```
не объявлена
   │ pan add <источник>
   ▼
объявлена в манифесте, не разрешена
   │ разрешение графа (pan add / pan install)
   ▼
разрешена (запись в pan.lock, копия в модули/)  ──┐
   │ pan remove <имя>                              │ pan install (lock не изменился)
   ▼                                                │ идемпотентно, без сети (SC-004)
удалена из манифеста и lock, модули/ очищена ◄──────┘
```

Конфликт версий/цикл — терминальное состояние попытки разрешения (FR-005):
процесс прерывается до записи в `pan.lock`, существующее состояние диска не
меняется (edge cases в spec.md).

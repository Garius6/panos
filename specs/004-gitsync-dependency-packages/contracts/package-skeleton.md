# Contract: структура пакета-заготовки

**Feature**: `004-gitsync-dependency-packages`

Применяется к каждому из 7 пакетов: `tempfiles`, `v8runner`, `gitrunner`,
`v8storage`, `cli`, `cli-selector`, `configor`. Все шаги идентичны для
каждого — только имя директории/пакета меняется.

## Шаги создания (для каждого из 7)

```bash
mkdir -p ../panosiki/<латинское-имя>
cd ../panosiki/<латинское-имя>
git init -q
panos /абсолютный/путь/до/panosiki/pan/start.ps init
# pan init создаёт pan.toml (имя=имя директории, версия=0.1.0, точка_входа=start.ps) и start.ps
git add .
git -c user.email=... -c user.name=... commit -qm "<русское сообщение коммита>"
git tag v0.1.0
```

**Важно**: `pan init` определяет имя пакета по имени ТЕКУЩЕЙ директории
(через `ос.выполнить("/bin/pwd", ...)`, см. `panosiki/pan/start.ps`), значит
директория ДОЛЖНА называться ровно так же, как латинское имя пакета
(`tempfiles`, не `tempfiles-pkg` и т.п.) — иначе `pan.toml` получит не то
`имя`, которое требует FR-002.

## Проверка (после создания каждого)

```bash
cat ../panosiki/<латинское-имя>/pan.toml   # имя = "<латинское-имя>", версия = "0.1.0"
git -C ../panosiki/<латинское-имя> log --oneline   # ровно 1 коммит
git -C ../panosiki/<латинское-имя> tag             # ровно "v0.1.0"
```

## Что НЕ входит в заготовку

- Никакой бизнес-логики зависимости (FR-003) — `start.ps` остаётся ровно
  таким, каким его создал `pan init`.
- Никаких зависимостей в `pan.toml` — `[зависимости]` пуст.
- Никакого удалённого git-remote — репозиторий только локальный на этом
  этапе (как уже существующие `panosiki/pan`, `panosiki/panos-raylib`).

## Конфликт имён

Если `panosiki/<латинское-имя>` уже существует (директория занята
чем-то другим) — остановиться и решить явно (переименование/алиас, FR-011),
не перезаписывать содержимое существующей директории.

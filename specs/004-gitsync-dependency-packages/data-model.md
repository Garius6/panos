# Data Model: Пакеты-заготовки для зависимостей gitsync

**Feature**: `004-gitsync-dependency-packages` | **Date**: 2026-07-21

Источник — раздел "Key Entities" в `spec.md`. Эта фича не вводит новых
рантайм-типов данных panos (в отличие от 003) — "сущности" здесь means
артефакты на диске/в git, плюс одна карта-документ.

## Пакет-заготовка

Директория в `panosiki/`, полностью независимый git-репозиторий.

| Поле | Значение | Источник |
|------|----------|----------|
| Расположение | `panosiki/<латинское-имя>/` | плоско, без вложенности (как `panosiki/pan`) |
| `pan.toml` → `имя` | латинское, = оригинальному имени OneScript-зависимости | research.md п.2 |
| `pan.toml` → `версия` | `0.1.0` | research.md п.2 |
| `pan.toml` → `точка_входа` | `start.ps` (default `pan init`) | не переопределяется |
| `pan.toml` → `зависимости` | пусто | нет логики — нечему зависеть |
| Содержимое точки входа | ровно то, что генерирует `pan init` | FR-003 — без бизнес-логики |
| git | инициализирован, 1 коммит, тег `v0.1.0` | research.md п.3 |

**Валидация** (проверяется вручную/скриптом при выполнении задач):
- Директория существует, `pan.toml` парсится (`манифест.загрузить` из
  `panosiki/pan`).
- `git log` в директории содержит ровно 1 коммит; `git tag` содержит
  `v0.1.0` указывающий на этот коммит.
- Имя пакета не конфликтует с уже существующей директорией в `panosiki/`
  (FR-011).

**Экземпляры** (ровно 7, из Key Entities в spec.md):
`tempfiles`, `v8runner`, `gitrunner`, `v8storage`, `cli`, `cli-selector`,
`configor`.

## Модуль stdlib `слог`

Единственный файл `std/слог.ps` (не git-репозиторий, не пакет — обычный
файл стандартной библиотеки panos, наравне с `std/архив.ps`,
`std/супервизор.ps`).

| Функция | Сигнатура | Поведение |
|---------|-----------|-----------|
| `отладка` | `(текст: Строка) -> Пусто` | печатает `[ОТЛАДКА] текст` |
| `инфо` | `(текст: Строка) -> Пусто` | печатает `[ИНФО] текст` |
| `предупреждение` | `(текст: Строка) -> Пусто` | печатает `[ПРЕДУПРЕЖДЕНИЕ] текст` |
| `ошибка` | `(текст: Строка) -> Пусто` | печатает `[ОШИБКА] текст` |
| `критично` | `(текст: Строка) -> Пусто` | печатает `[КРИТИЧНО] текст` |

Вывод — всегда через `ввод_вывод.печать`/`ввод_вывод.строка` в stdout, без
фильтрации по уровню, без appenders/layouts (research.md п.4-5). Не имеет
собственного persisted-состояния.

## Отложенная зависимость

Не артефакт на диске — запись в карте соответствия (см. ниже) со статусом
"отложена". Единственный текущий экземпляр: `reflector`.

| Поле | Значение |
|------|----------|
| Причина блокировки | конкретная отсутствующая возможность panos (не общая формулировка) |
| Что нужно для снятия блокировки | описано, но НЕ реализуется в этой фиче |

## Карта соответствия

Не отдельный файл — таблица в разделе "Key Entities" `spec.md` (US3).
15 строк, по одной на каждую исходную рантайм-зависимость `gitsync`.

| Поле | Тип | Описание |
|------|-----|----------|
| Оригинал (версия) | Строка | имя и версия у OneScript |
| Судьба на panos | Перечисление | новый пакет / частичное покрытие / вошла в stdlib / исключена / отложена |
| Латинское имя пакета | Строка \| — | для "новый пакет"/"частичное покрытие" |
| Русский алиас | Строка \| — | рекомендация для будущих `pan add ... <алиас>` |
| Назначение / примечание | Строка | однострочное описание + при необходимости — что покрывает пересечение или что блокирует перенос |

**Инвариант**: ровно 15 строк, ровно одна на каждую исходную зависимость;
сумма по статусам = 7 (новый пакет: 6 + частичное покрытие: 1 `configor`)
+ 6 (исключена) + 1 (вошла в stdlib) + 1 (отложена) = 15.

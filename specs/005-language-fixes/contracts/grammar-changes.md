# Grammar Contract: Устранение проблем языка panos

**Feature**: `005-language-fixes` | **Date**: 2026-07-23

Компилятор panos не публикует API в обычном смысле — его "контракт" с
разработчиком panos-пакетов — это грамматика языка (что компилируется/что
нет) и сообщения об ошибках. Ниже — точная граница поведения ДО/ПОСЛЕ для
каждой из трёх правок; это то, что должно проверяться e2e-тестами
(`core/e2e_language_fixes_test.odin`, см. quickstart.md).

## P1 — квалифицированный generic-тип

**До**:

```panos
импорт "модуль" как м

тип Коробка[T] = структура
    значение: T
конец
```

```panos
// в другом файле, импортирующем модуль выше
функ построить() -> м.Коробка(Число)   // Resolve Error: undefined variable 'Число'
    ...
конец
```

**После**: та же программа компилируется без ошибок; `м.Коробка(Число)`
резолвится в конкретный инстанцированный тип, эквивалентный тому, что дал
бы локальный `Коробка(Число)`, будь тип `Коробка` объявлен в том же файле.

**Границы**:
- Неэкспортированный тип по-прежнему ошибка ("модуль не экспортирует...").
- Несовпадение числа аргументов типа — `Type Error`, как для локальных
  generic-типов.
- НЕ-generic квалифицированный тип (`м.Тип` без `(...)`) — поведение не
  меняется вообще.

## P2 — многострочная ветка `выбор`

**До**:

```panos
выбор значение
    Вариант(x) ->
        это.список.добавить(x)      // Синтаксическая ошибка: ожидалось Arrow
        Результат.Успех(x)
конец
```

**После**: добавляется вторая, явно маркированная форма ветки —

```panos
выбор значение
    Вариант(x) тогда
        это.список.добавить(x)
        Результат.Успех(x)
    конец
конец
```

**Границы**:
- Однострочная форма (`Вариант(x) -> Результат.Успех(x)`) — без изменений,
  продолжает работать буквально как раньше.
- В многострочной форме значение ветки — тип ПОСЛЕДНЕГО выражения блока
  (как в теле функции); типы веток одного `выбор` должны совпадать по
  этому хвостовому значению — иначе `Type Error: ветки выбора возвращают
  разные типы`, тот же диагностический путь, что уже есть сегодня.
- `возврат` посреди многострочной ветки — обычная семантика раннего
  выхода из функции, не требует хвостового выражения после себя (как в
  любом другом блоке этого языка).
- Смешивать `->` и `тогда...конец` в РАЗНЫХ ветках ОДНОГО `выбор` —
  разрешено (каждая ветка выбирает форму независимо).

## P3 — завершающая запятая

**До** (для затронутых конструкций — список аргументов вызова, список
типов варианта перечисления, список аргументов шаблона-конструктора,
список type-аргументов generic-типа, список элементов tuple-типа):

```panos
функция(
    аргумент1,
    аргумент2,
)   // Синтаксическая ошибка: неожиданный токен ')'
```

**После**: та же запись компилируется идентично записи без завершающей
запятой.

**Границы**:
- Допускается РОВНО одна завершающая запятая, и только если список уже
  содержит хотя бы один элемент (`функция(,)` остаётся ошибкой).
- Списки, УЖЕ допускавшие завершающую запятую (параметры функции при
  объявлении, литералы `массив()`/`соответствие()`, параметры
  функционального типа) — поведение не меняется.

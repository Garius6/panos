
## Ручная проверка P1 — квалифицированный generic-тип

```sh
mkdir -p /tmp/lang_fixes_check && cd /tmp/lang_fixes_check
cat > a.ps <<'EOF'
экспорт тип Коробка[T] = структура
    значение: T
конец
EOF
cat > b.ps <<'EOF'
импорт "a.ps" как м
импорт "ввод_вывод" как ио

функ построить() -> м.Коробка(Строка)
    м.Коробка("привет")
конец

функ старт() -> Пусто
    пер к = построить()
    ио.строка(к.значение)
конец
EOF
/path/to/panos b.ps   # ожидается: печатает "привет", без ошибок резолва/типов
```

## Ручная проверка P2 — многострочная ветка `выбор`

```sh
cat > /tmp/match_block.ps <<'EOF'
импорт "ввод_вывод" как ио

функ проверить(значение: Опция(Число)) -> Число
    выбор значение
        Опция.Есть(x) тогда
            ио.строка("получено")
            x + 1
        конец
        Опция.Нет -> 0
    конец
конец

функ старт() -> Пусто
    ио.строка(строки.из_числа(проверить(Опция.Есть(41))))
конец
EOF
/path/to/panos /tmp/match_block.ps   # ожидается: печатает "получено", затем "42"
```

## Ручная проверка P3 — завершающая запятая

```sh
cat > /tmp/trailing_comma.ps <<'EOF'
импорт "ввод_вывод" как ио

функ сложить(а: Число, б: Число) -> Число
    а + б
конец

функ старт() -> Пусто
    ио.строка(строки.из_числа(сложить(
        1,
        2,
    )))
конец
EOF
/path/to/panos /tmp/trailing_comma.ps   # ожидается: печатает "3", без синтаксической ошибки
```

## Новые e2e-тесты

Все три сценария выше (плюс граничные случаи из spec.md → Edge Cases и
contracts/grammar-changes.md → Границы) кодируются как `core/
e2e_language_fixes_test.odin`, использующий тот же `run_code`-хелпер, что
и существующие `core/e2e_*_test.odin` файлы. Прогоняется как часть `just
test` — отдельной команды не требуется.

# CLI Contract: gitsync

**Feature**: `006-gitsync-port` | **Date**: 2026-07-23

Единственный внешний интерфейс этого пакета — командная строка (запускается
как `pan task gitsync ...` или напрямую `panos start.ps ...` через
`panosiki/cli`, флаги/подкоманды которого уже реализованы и протестированы).

## `gitsync sync [ОПЦИИ] ПУТЬ_ХРАНИЛИЩА РАБОЧИЙ_КАТАЛОГ`

Синоним: `gitsync s ...`

| Опция | Синоним | Переменная окружения | По умолчанию |
|-------|---------|----------------------|--------------|
| `--storage-user` | `-u` | `GITSYNC_STORAGE_USER` | `Администратор` |
| `--storage-pwd` | `-p` | `GITSYNC_STORAGE_PASSWORD` | `""` |
| `--ext` | `-e` | `GITSYNC_EXTENSION` | `""` (основная конфигурация) |

Позиционные (после разбора опций):
- `ПУТЬ_ХРАНИЛИЩА` — обязателен, либо аргументом, либо `GITSYNC_STORAGE_PATH`.
- `РАБОЧИЙ_КАТАЛОГ` — необязателен, по умолчанию текущий каталог, либо
  `GITSYNC_WORKDIR`.

**Поведение**: см. spec.md User Story 1 / FR-001..FR-005. Код возврата `0`
при успехе (включая "нечего синхронизировать" — FR-004), ненулевой при
ошибке (не git-репозиторий, ошибка платформы/хранилища на середине
версии — VERSION НЕ обновляется, см. Edge Cases).

## `gitsync init [ОПЦИИ] ПУТЬ_ХРАНИЛИЩА [РАБОЧИЙ_КАТАЛОГ]`

**Поведение**: см. spec.md User Story 2 / FR-006. Идемпотентен — повторный
вызов на уже инициализированном каталоге не трогает существующие
`VERSION`/`AUTHORS`.

## `gitsync set-version НОМЕР [РАБОЧИЙ_КАТАЛОГ]`

Синоним: `gitsync sv ...`

**Поведение**: см. spec.md User Story 3 / FR-007. `НОМЕР` — обязательный
позиционный аргумент, целое неотрицательное; иное — ошибка, `VERSION` не
изменяется.

## Общие переменные окружения (не привязаны к одной подкоманде)

| Имя | Описание |
|-----|----------|
| `GITSYNC_V8_PATH` | путь к исполняемому файлу платформы 1С (`v8runner`/`v8storage`.`установить_путь_к_платформе`) |
| `GITSYNC_TEMP` | каталог временных файлов (`tempfiles.Менеджер`, если не системный по умолчанию) |

Не портируются в v1 (см. spec.md Assumptions): `GITSYNC_V8VERSION` (выбор
версии платформы по маске — не нужен, когда путь к исполняемому файлу
задаётся явно), `GITSYNC_VERBOSE` (логирование — `слог` уже покрывает
эту потребность на уровне кода, отдельный флаг многословности не входит
в v1), `GITSYNC_EMAIL` (домен почты по умолчанию — заменяется явным
`email` в `AUTHORS`, см. data-model.md).

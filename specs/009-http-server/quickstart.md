# Quickstart: HTTP-сервер panos

## Ручная проверка (реальный curl, реальный сокет — не 1С-специфика, ничего не подделывается)

```panos
// server.ps
импорт сеть
импорт "ввод_вывод" как ио

функ старт() -> Пусто
    пер слушатель_рез = сеть.http_сервер_слушать(8080)
    если слушатель_рез.ошибка() тогда
        ио.строка("не смог начать слушать: " + слушатель_рез.причина().сообщение)
        возврат
    конец
    пер слушатель = слушатель_рез.значение()

    ио.строка("слушаю :8080")
    пока истина цикл
        пер запрос_рез = слушатель.принять_запрос()
        если запрос_рез.ошибка() тогда
            продолжить
        конец
        пер запрос = запрос_рез.значение()
        ио.строка(запрос.метод() + " " + запрос.путь())
        запрос.ответить(200, "text/plain", "привет от panos: " + запрос.путь())
    конец
конец
```

```sh
panos server.ps &
sleep 0.5
curl -s -i http://127.0.0.1:8080/test
# ожидается: HTTP/1.1 200 OK, Content-Type: text/plain, тело "привет от panos: /test"
kill %1
```

## Проверка параллельности (Acceptance Scenario 3)

```panos
// server_concurrent.ps — 3 процесса на один слушатель
импорт сеть
импорт "ввод_вывод" как ио

функ обработчик(слушатель: Слушатель) -> Пусто
    пока истина цикл
        пер запрос_рез = слушатель.принять_запрос()
        если не запрос_рез.ошибка() тогда
            пер запрос = запрос_рез.значение()
            запрос.ответить(200, "text/plain", "обработано")
        конец
    конец
конец

функ старт() -> Пусто
    пер слушатель = сеть.http_сервер_слушать(8080).значение()
    запусти обработчик(слушатель)
    запусти обработчик(слушатель)
    обработчик(слушатель) // старт() сам — третий обработчик, а не только спавнер:
                          // после запусти() он иначе завершился бы немедленно
                          // (VM выходит, как только процесс 0 завершается, не
                          // дожидаясь спавненных детей) и оборвал бы сервер.
конец
```

```sh
panos server_concurrent.ps &
sleep 0.5
for i in 1 2 3 4 5; do curl -s http://127.0.0.1:8080/ & done
wait
# ожидается: все 5 curl получают "обработано", ни один не завис
kill %1
```

## Новые/изменённые тесты

- `odin test ./core` — модульные тесты моста там, где не нужен реальный
  сокет (например, преобразование `Bridge_Request` -> `Http_Request_
  Value`, повторный `.ответить()` -> `Результат.Неудача`).
- Ручной прогон (этот файл) — единственный способ честно проверить
  реальный accept/respond цикл целиком, curl как внешний независимый
  клиент.
